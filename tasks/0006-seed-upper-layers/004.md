# 004: Seed prompt templates

## Context

Each upper-layer seed agent needs a prompt that tells it what to create. Seed prompts are distinct from cascade generator prompts in framing: they instruct the agent to "create the initial set from scratch" rather than "review and update the existing set." Seed prompts receive only the content from layers below — no existing layer content, no feedback memos, no cross-layer context.

Three new prompt template files are needed, one per upper layer. They live alongside the existing values and SA seed prompts in the `prompts/seed/` directory.

This step also requires updating the prompt content validation test to register the new template files and their required variables, or that test will fail with a coverage gap assertion.

## Reuse Opportunities

- The quality standards, analytical expectations, and output format documentation from each generator prompt should be drawn on heavily. Each seed prompt should demand the same rigor as its generator counterpart — same time horizons, same trade-off honesty, same specificity requirements, same frontmatter fields. The generator prompts are the authoritative source for "what good output looks like" for each layer.

- The SA seed prompt (`seed.md`) is the structural template for seed-specific concerns: how to frame bootstrapping (creating from scratch vs. updating), how to instruct the agent to use `write_file` to create items, how to request a README.md narrative summary, how to reference the current date for timestamps.

- The `last_modified_by` naming convention follows the existing pattern across the codebase. Generators use `{role}-generator` (e.g., `strategic-generator`). The SA seed uses `seed-agent`. The new seed prompts should use `{role}-seed-agent` (e.g., `strategic-seed-agent`) to distinguish seed-created items from cascade-generated items.

## Deliverables

### Strategic objectives seed prompt

A markdown template file at `src/policy_factory/prompts/seed/strategic.md` with two template variables: `current_date` and `context_below`.

The prompt must cover:

- **Role and task framing** — The agent is creating the initial Strategic Objectives layer from scratch. Explain the 5-layer model and where strategic objectives sit (layer 3: bridging values and action).
- **Context injection** — The `context_below` variable contains formatted content from the values and SA layers. The prompt must explain that strategic objectives must be grounded in both value tensions AND situational reality.
- **Quality standards** — Drawn from the generator prompt: 5-15 year time horizon, must reference which value tensions each objective addresses, must be grounded in specific situational assessments, must address genuine trade-offs (state what gets cut or deprioritized), must be measurable or assessable, must be willing to state uncomfortable strategic positions. Include the same calibration examples for directness.
- **Output format** — YAML frontmatter with `title`, `status` (active), `created`, `last_modified`, `last_modified_by` (set to `strategic-seed-agent`), and `references` to items in layers below. Body content describing the objective.
- **Tool usage instructions** — Use `write_file` to create markdown files in the `strategic-objectives/` directory. Also create a `strategic-objectives/README.md` narrative summary tying all objectives together.
- **Volume guidance** — Target 6-10 strategic objectives.
- **Current date** — Available as `current_date` for timestamps in frontmatter.

Key differences from the generator prompt: No `layer_content`, `feedback_memos`, or `cross_layer_context` variables. No instructions to read existing files, incorporate feedback, or delete stale items. Pure bootstrapping — the layer is empty and the agent creates everything fresh.

### Tactical objectives seed prompt

A markdown template file at `src/policy_factory/prompts/seed/tactical.md` with the same two template variables: `current_date` and `context_below`.

Same structural pattern as the strategic seed prompt, but with tactical-specific requirements drawn from the tactical generator prompt:

- 1-5 year time horizon
- Must reference the strategic objective(s) each tactic serves
- Must name specific Finnish institutional actors and responsible bodies (which ministry, agency, or body owns this)
- Must include success metrics with numbers where possible
- Must estimate resource requirements (budget, personnel, institutional capacity)
- Must be honest about political difficulty — rate it, name which factions oppose, state their strongest arguments
- `last_modified_by` set to `tactical-seed-agent`
- Target directory: `tactical-objectives/`
- Must create `tactical-objectives/README.md` narrative summary
- Volume guidance: target 8-15 tactical objectives (more granular than strategic)

The `context_below` for this prompt will contain content from values, SA, AND strategic objectives (all three layers below tactical).

### Policies seed prompt

A markdown template file at `src/policy_factory/prompts/seed/policies.md` with `current_date` and `context_below`.

Policies-specific requirements drawn from the policies generator prompt:

- Must reference the tactical objective(s) each policy implements
- Must produce the full detailed policy document structure from the generator prompt: Summary, Rationale, Implementation, Cost Estimate, Political Feasibility, International Context, Risks
- Must be specific enough for policymakers to act on — mechanisms, costs, timelines, responsible institutions
- Must assess political feasibility honestly — name parties and interest groups
- Must consider international alignment (EU law, Nordic cooperation, NATO obligations)
- Must include honest "would this actually work?" assessment with failure modes
- `last_modified_by` set to `policies-seed-agent`
- Target directory: `policies/`
- Must create `policies/README.md` narrative summary
- Volume guidance: target 10-20 policy recommendations (most granular layer)

The `context_below` for this prompt will contain content from all 4 layers below (values, SA, strategic, tactical).
