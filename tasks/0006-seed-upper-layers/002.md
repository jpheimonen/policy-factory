# 002: Extract shared context gathering

## Context

The seed endpoints for the upper layers need two capabilities that partially exist already:

1. **Gathering content from all layers below** a target layer — the orchestrator's private `_gather_generation_context()` does this, and the SA seed endpoint has its own inline version that only reads the values layer. Both need to be replaced by a shared utility.

2. **Prerequisite validation** — checking that all layers below a target have at least one item before allowing seeding. This doesn't exist anywhere yet.

The good news: the building blocks are already in place. `layers_below()` is a public function exported from `policy_factory.cascade` that returns the list of layer slugs below a given layer. `gather_layer_content()` in `policy_factory.cascade.content` gathers the full content (narrative + items) of a single layer. The new helper composes these two existing functions to gather content from ALL layers below.

This step is prerequisite for step 005 (upper-layer seed endpoints), which will call both helpers.

## Related Code

Existing files to modify:

- `src/policy_factory/cascade/content.py` — Already contains `gather_layer_content()` and `gather_cross_layer_context()`. This is the natural home for the new `gather_context_below()` function, since it's a content-gathering utility that composes existing functions from the same module and the orchestrator.

- `src/policy_factory/cascade/__init__.py` — Already exports `gather_layer_content`, `gather_cross_layer_context`, `layers_below`, `layers_from`. The new `gather_context_below` and `check_prerequisites` functions need to be added to the exports.

- `src/policy_factory/cascade/orchestrator.py` — The private `_gather_generation_context()` function (line 159) duplicates the logic that `gather_context_below()` will provide. After the new function exists, `_gather_generation_context()` should delegate to it rather than reimplementing the layer iteration. The `user_context` parameter is specific to generation (not used by seed), so `_gather_generation_context()` becomes a thin wrapper that calls `gather_context_below()` and appends the user context.

- `tests/test_cascade_content.py` — Existing tests for `gather_layer_content` and `gather_cross_layer_context`. New tests for `gather_context_below` and `check_prerequisites` should be added here, following the existing test structure with the `data_dir` and `populated_data_dir` fixtures.

- `tests/test_cascade_orchestrator.py` — The existing `TestContextGathering` class tests `_gather_generation_context`. After the refactor, these tests should still pass since the function's behavior doesn't change — only its implementation delegates to the new shared utility.

## Reuse Opportunities

- `gather_layer_content()` already handles all the complexity of reading narratives and items from a single layer, including error handling for unreadable items. The new `gather_context_below()` just loops over `layers_below()` and calls it for each layer.

- The existing `populated_data_dir` fixture in `test_cascade_content.py` creates values and SA content. It needs extending with strategic-objectives content for testing multi-layer context gathering.

- `list_items()` from `policy_factory.data.layers` is the right function for prerequisite validation — it returns a list of `ItemSummary` objects, so checking `len(list_items(data_dir, slug)) > 0` is the prerequisite test.

## Deliverables

### New function: `gather_context_below()`

Add to `src/policy_factory/cascade/content.py`:

```python
def gather_context_below(
    data_dir: Path,
    layer_slug: str,
) -> str:
```

This function:
- Calls `layers_below(layer_slug)` to get the ordered list of layers below the target.
- For each layer below, calls `gather_layer_content(data_dir, below_slug)` to get its content.
- Wraps each layer's content under a heading with the layer's display name (e.g., `## Values Layer`).
- Joins all sections with newlines and returns the assembled string.
- Returns an empty string if there are no layers below (bottom layer case).
- Raises `ValueError` if the layer slug is invalid (propagated from `layers_below()`).

The output format should match what `_gather_generation_context()` currently produces so that existing prompt templates continue to work:
```
## Values Layer

[content from gather_layer_content]

## Situational Awareness Layer

[content from gather_layer_content]
```

### New function: `check_prerequisites()`

Add to `src/policy_factory/cascade/content.py`:

```python
def check_prerequisites(
    data_dir: Path,
    layer_slug: str,
) -> list[str]:
```

This function:
- Calls `layers_below(layer_slug)` to get the required layers.
- For each layer below, calls `list_items(data_dir, slug)` to check if it has any items.
- Returns a list of layer slugs that are empty (have zero items). An empty return list means all prerequisites are met.
- Does NOT check the narrative/README — only items count for prerequisite purposes.
- Raises `ValueError` if the layer slug is invalid (propagated from `layers_below()`).

### Refactor `_gather_generation_context()`

Update the private function in `orchestrator.py` to delegate to `gather_context_below()`:

```python
def _gather_generation_context(
    data_dir: Path,
    layer_slug: str,
    user_context: str | None = None,
) -> str:
    parts: list[str] = []

    below_content = gather_context_below(data_dir, layer_slug)
    if below_content:
        parts.append(below_content)

    if user_context:
        parts.append(f"## User Input\n{user_context}\n")

    return "\n".join(parts)
```

The function keeps its existing signature and behavior — it's still private and still supports `user_context`. The only change is that the layer iteration and content gathering is now done by the shared utility.
