# 003: Rework agent config for SDK tool resolution

## Context

The config module currently maps agent roles to Anthropic API format tool definition dicts. With the SDK switch, tools are provisioned via an MCP server and controlled via `allowed_tools` string lists in `ClaudeAgentOptions`. The config module needs to express per-role tool access in terms that `AgentSession` can use to build `ClaudeAgentOptions`.

There are two separate concerns per role:
1. **Which MCP tools to include in the server** â€” determines what tools the Claude Code CLI process can actually invoke (full file tools, read-only subset, or none)
2. **What goes in the `allowed_tools` list** â€” determines what Claude is permitted to use (MCP server reference, `"WebSearch"`, or empty)

These replace the old `resolve_tools()` which returned raw Anthropic API tool dicts.

This step also changes `AgentConfig` to carry the role, so `AgentSession` (step 004) can resolve both the MCP tool set and the allowed_tools list internally.

## Related Code

Existing files to modify:

- `src/policy_factory/agent/config.py` â€” The main file being reworked. Contains `AgentRole` literal, `_DEFAULT_MODELS`, `_ENV_VAR_MAP`, `resolve_model()` (stays unchanged), `_TOOLS_BY_ROLE` (to be reworked), `resolve_tools()` (to be replaced), and `AgentConfig` dataclass (to be modified).
- `tests/test_agent_config.py` â€” Contains `TestResolveModel` (stays unchanged), `TestAgentConfig` (to be updated), and `TestResolveTools` (to be replaced).
- `src/policy_factory/agent/__init__.py` â€” Currently exports `resolve_tools`. Must be updated to export the replacement function.

Reference files:

- `src/policy_factory/agent/tools.py` (after step 002) â€” Provides the MCP tool objects and the server factory function. The config module needs to know which tool objects exist and how to select subsets, but should not import the raw tool objects for the per-role mapping â€” that mapping is expressed in terms of a tool set enum or identifier that the server factory understands.

Caller patterns to be aware of (updated in step 005, but the interface designed here must support them):

- Some callers pass `tools=resolve_tools(role)` to `AgentConfig` â€” `seed.py` does this for both `values-seed` and `seed` roles.
- Other callers pass only `model=model` to `AgentConfig` and don't pass tools at all â€” the cascade orchestrator, critic_runner, classifier, heartbeat orchestrator, ideas generator, and ideas evaluator all do this.
- After this step, callers should be able to pass a `role` to `AgentConfig` and have tool resolution happen internally.

## Reuse Opportunities

- **`resolve_model()` â€” 100% reuse**: The model resolution function, `_DEFAULT_MODELS`, `_ENV_VAR_MAP`, and `AgentRole` literal all stay exactly as they are.
- **Per-role tool mapping structure**: The current `_TOOLS_BY_ROLE` dict encodes the correct per-role tool access logic. The same mapping needs to be preserved, just expressed differently â€” instead of mapping to Anthropic tool dicts, it maps to a tool set identifier (full, read-only, none) and a boolean for web search.
- **`resolve_tools()` validation pattern**: The unknown-role check with `ValueError` and descriptive message should be preserved in the replacement function.

## Deliverables

### AgentConfig dataclass changes

The `AgentConfig` dataclass is updated:

- **Remove** the `tools: list[dict[str, Any]]` field
- **Add** a `role: str | None` field (defaulting to `None`) that identifies the agent's role for tool resolution
- Keep `model: str | None` and `system_prompt: str | None` unchanged
- Update the docstring to reflect that tool provisioning is now role-based via the SDK

When `role` is set, `AgentSession` (step 004) will use it to determine the MCP tool set and `allowed_tools` list. When `role` is `None`, no tools are provisioned (equivalent to an empty tool set).

### New tool resolution function

Replace `resolve_tools()` with a new function (e.g., `resolve_allowed_tools(role)`) that returns the `allowed_tools` string list for `ClaudeAgentOptions`. The per-role logic:

- **generator**: MCP server reference only (full file tools, no web search)
- **critic**: MCP server reference only (read-only file tools, no web search)
- **synthesis**: empty list (no tools)
- **classifier**: empty list (no tools)
- **heartbeat-skim**: `"WebSearch"` only (no MCP file tools)
- **heartbeat-triage**: `"WebSearch"` only (no MCP file tools)
- **heartbeat-sa-update**: MCP server reference + `"WebSearch"` (full file tools and web search)
- **seed**: MCP server reference + `"WebSearch"` (full file tools and web search)
- **values-seed**: empty list (no tools)
- **idea-evaluator**: empty list (no tools)
- **idea-generator**: empty list (no tools)

The MCP server reference string follows the `mcp__<server-name>` convention. Since step 002 uses `"policy-factory-tools"` as the server name, the reference string is `"mcp__policy-factory-tools"`.

The function must:
- Raise `ValueError` for unknown roles (same validation pattern as current `resolve_tools`)
- Return a copy to prevent mutation of internal state

### New MCP tool set resolution function

A companion function (e.g., `resolve_tool_set(role)`) that determines which MCP tools to include in the MCP server for a given role. This returns a tool set identifier that the server factory (from step 002) understands:

- **generator, heartbeat-sa-update, seed**: full file tools (all four)
- **critic**: read-only file tools (list_files and read_file only)
- **All other roles**: no file tools (no MCP server needed)

This function is used by `AgentSession` (step 004) to decide whether and how to create the MCP server.

The function must:
- Raise `ValueError` for unknown roles
- Return a value that clearly distinguishes "full tools", "read-only tools", and "no tools"

### Remove old tool imports and constants

- Remove the imports of `FILE_TOOLS`, `FILE_TOOLS_WITH_WEB_SEARCH`, `READ_ONLY_TOOLS`, `WEB_SEARCH_ONLY` from `tools.py` (these were removed in step 002)
- Remove the `_TOOLS_BY_ROLE` dict that mapped roles to Anthropic tool dicts
- Remove the `resolve_tools()` function
- Update the module docstring

### Update agent/__init__.py

- Remove `resolve_tools` from imports and `__all__`
- Add the new tool resolution function(s) to imports and `__all__`
- Update the module docstring
