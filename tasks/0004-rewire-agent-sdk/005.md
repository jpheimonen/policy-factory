# 005: Update all 8 caller modules

## Context

Steps 001–004 rewired the agent framework internals: the dependency was swapped, tools were converted to MCP, config was reworked for role-based resolution, and `AgentSession` was rewritten for `ClaudeSDKClient`. But the 8 modules that actually *use* the agent framework still reference the old Anthropic client and the old `AgentConfig` construction pattern. They will not compile or work until updated.

This step is straightforward and mechanical — every caller undergoes the same transformation pattern. Each change is small (remove an import, remove a function call, remove a parameter, update the config construction), but there are 8 modules to update, each with slightly different context around the shared pattern.

The changes are pure removals and simplifications. No new behavior is introduced. The callers continue to work exactly as before — they just construct `AgentSession` and `AgentConfig` differently.

## Related Code

Caller modules to modify (all 8):

- `src/policy_factory/heartbeat/orchestrator.py` — The `_run_tier_agent()` helper function (lines 149–283) is the single call site. It imports `AgentConfig`, `resolve_model`, and `AgentSession` from the agent module, and `get_anthropic_client` from server deps. It constructs `AgentConfig(model=model)` without tools and passes `client=client` to `AgentSession`. The `get_anthropic_client()` call and `client=client` parameter need removal. The `AgentConfig` construction needs to gain the role. The `agent_type` string passed to `store.create_agent_run()` already matches the agent role slug.

- `src/policy_factory/cascade/orchestrator.py` — The `_default_generation_runner()` function (lines 208–288) is the call site. Same pattern: imports `get_anthropic_client`, calls it, passes `client=client` to `AgentSession`, constructs `AgentConfig(model=model)` without role. The role here is `"generator"`.

- `src/policy_factory/cascade/critic_runner.py` — The `_run_single_critic()` function (lines 162–358) is the call site. Same pattern. The role here is `"critic"`.

- `src/policy_factory/cascade/synthesis_runner.py` — The `run_synthesis()` function (lines 165–402) is the call site. It imports `get_anthropic_client` inline (line 300) rather than at the function's top import block. The role here is `"synthesis"`.

- `src/policy_factory/cascade/classifier.py` — The `classify_input()` function (lines 127–224) is the call site. Same pattern. The role here is `"classifier"`.

- `src/policy_factory/server/routers/seed.py` — Two separate call sites:
  - `seed_values()` endpoint (lines 208–370): Imports `resolve_tools` in addition to the usual imports. Calls `resolve_tools("values-seed")` and passes `tools=tools` to `AgentConfig`. Also wraps `get_anthropic_client()` in a `try/except RuntimeError` block that returns HTTP 503 on failure. The role here is `"values-seed"`.
  - `trigger_seed()` endpoint (lines 373–544): Same pattern as `seed_values()` but for role `"seed"`. Also calls `resolve_tools("seed")` and has the `try/except RuntimeError` guard.
  Both endpoints need the `resolve_tools` import removed, the `tools=tools` parameter replaced with the role, and the `try/except RuntimeError` guard removed.

- `src/policy_factory/ideas/generator.py` — The `generate_ideas()` function (lines 28–161) is the call site. Same pattern. The role here is `"idea-generator"`.

- `src/policy_factory/ideas/evaluator.py` — The `_run_scoring()` function (lines 171–275) is the call site. Same pattern. The role here is `"idea-evaluator"`.

Config interface from step 003 (what callers will use after this step):

- `AgentConfig` now accepts a `role` parameter (replacing `tools`). The session uses this to resolve both MCP tools and allowed_tools internally.
- `resolve_model()` stays unchanged — callers still call it the same way.
- `resolve_tools()` is gone — callers no longer call it.

Session interface from step 004 (what callers will use after this step):

- `AgentSession.__init__` no longer accepts `client`. Parameters are: `config`, `emitter`, `context_id`, `agent_label`, `data_dir`.
- `AgentSession.run()` returns `AgentResult` with the same fields plus `session_id`.
- `AgentResult.full_output` and `AgentResult.result_text` work the same as before.

## Reuse Opportunities

The transformation is identical across all 8 callers, varying only in the role string and surrounding context. The pattern for each caller:

1. **Remove** the `from policy_factory.server.deps import get_anthropic_client` import (or the inline import within the function body).
2. **Remove** the `client = get_anthropic_client()` call.
3. **Remove** the `client=client` parameter from the `AgentSession(...)` constructor call.
4. **Add** the role to the `AgentConfig(...)` constructor call (the specific role string matches the `agent_type` that each caller already passes to `store.create_agent_run()`).
5. For `seed.py` only: **remove** the `resolve_tools` import and the `tools=resolve_tools(role)` call, and the `tools=tools` parameter from `AgentConfig()`. Also **remove** the `try/except RuntimeError` guard around `get_anthropic_client()`.

All other code in each caller (prompt building, agent run recording, result handling, error handling, event emission) stays unchanged.
