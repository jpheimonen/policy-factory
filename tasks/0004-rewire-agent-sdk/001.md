# 001: Swap dependency and remove Anthropic infrastructure

## Context

This is the foundational cleanup step. Before the core agent components can be rewritten (steps 002–004), the Anthropic SDK dependency must be swapped out and all infrastructure that exists solely to support it must be removed. This includes the shared `AsyncAnthropic` client lifecycle in the server layer, the meditation filter (which is tied to the old streaming model), and the meditation preamble in prompt construction.

These removals are safe to do first because nothing in the remaining steps depends on the Anthropic client lifecycle or the meditation filter. The callers still reference `get_anthropic_client()` at this point — those references break at import time until step 005 updates them, but the test suite is expected to be in a transitional state across steps 001–006.

## Related Code

Existing files to modify:

- `pyproject.toml` — Contains `"anthropic>=0.39.0"` in the dependencies list (line 14). Replace with `"claude-agent-sdk==0.1.37"`.
- `src/policy_factory/server/deps.py` — Contains the Anthropic client lifecycle: `_anthropic_client` global, `_anthropic_client_initialized` flag, `init_anthropic_client()`, `shutdown_anthropic_client()`, `get_anthropic_client()`, and the `AsyncAnthropic` type import in the `TYPE_CHECKING` block. All of these must be removed. Everything else in this file (store, ws_manager, event_emitter, data_dir, scheduler, cascade controllers, auth dependencies) stays unchanged.
- `src/policy_factory/server/app.py` — The lifespan function calls `init_anthropic_client()` on startup (line 83) and `await shutdown_anthropic_client()` on shutdown (line 121). Both calls and their imports (lines 21, 24) must be removed. The comment on line 82 referencing "Anthropic client" should also be removed.
- `src/policy_factory/agent/meditation_filter.py` — The entire module (135 lines) is deleted.
- `tests/test_meditation_filter.py` — The entire test file is deleted.
- `src/policy_factory/agent/prompts.py` — `build_agent_prompt()` currently loads the meditation section and prepends it with a separator. Simplify to just call `load_prompt(category, name, **variables)` directly. Remove the `_SECTION_SEPARATOR` constant and the `load_section` import (unless other code uses it — check before removing).
- `src/policy_factory/agent/__init__.py` — Remove the `MeditationFilter` import and its entry in `__all__`. The `FILE_TOOLS`, `READ_ONLY_TOOLS`, and `TOOL_FUNCTIONS` exports will be removed in step 002 when the tool definitions change — do NOT remove them here as `config.py` still references them.
- `tests/test_agent_prompts.py` — All tests that assert meditation content is present must be updated to assert it is absent. The `TestBuildAgentPromptWithRealPrompts` tests that check for "10" and "bias" (meditation markers) need to be changed to verify the template loads without meditation. The fixture that creates a `sections/meditation.md` temp file is no longer needed for prompt construction tests.

## Reuse Opportunities

- The `load_prompt()` function from `policy_factory.prompts` already handles template loading with variable substitution. After removing the meditation wrapper, `build_agent_prompt()` becomes a thin pass-through to `load_prompt()` — it may be worth keeping as a convenience to preserve the existing call sites, or it can be simplified to a one-line delegation.
- The cc-runner project at `/home/jp/projects/modernpath/cc-runner/pyproject.toml` shows the exact dependency line for `claude-agent-sdk==0.1.37`.
