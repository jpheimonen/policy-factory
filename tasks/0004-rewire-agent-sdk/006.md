## Reuse Opportunities

The four caller test files all need the same category of change: remove the `get_anthropic_client` patch, simplify to a single `AgentSession` patch. The transformation is identical in concept across all four files, differing only in how each file structures its patches:

1. **`create_mock_agent_patches()` helper** â€” used in `test_heartbeat_orchestrator.py` and `test_critic_runner.py`. Both files define this identically. The helper currently returns a 2-tuple `(session_patch, client_patch)`. After the change, it returns a single patch context manager. All call sites that do `p1, p2 = create_mock_agent_patches(mock_run)` followed by `with p1, p2:` simplify to `p = create_mock_agent_patches(mock_run)` followed by `with p:`. The helper's docstring updating is the same in both files.

2. **`mock_agent_session()` / `mock_agent_session_with_side_effect()` context managers** â€” used in `test_integration_seeding.py`. Both currently nest two `with patch(...)` blocks. Each simplifies to a single `with patch(...)` block. The `mock_client = MagicMock()` line and its corresponding patch are removed from both.

3. **Inline `patch()` blocks** â€” used in `test_classifier.py`. The `with patch(...), patch(...), patch(...), patch(...):` chains in both test methods lose one `patch()` call (the `get_anthropic_client` one).

4. **`MockAgentResult` dataclass** â€” defined identically in `test_heartbeat_orchestrator.py` and `test_critic_runner.py`. Both need the same `session_id` field addition.

5. **`AgentResult` construction** â€” `test_integration_seeding.py` constructs real `AgentResult` instances. All three helper functions (`_make_mock_values_agent_result`, `_make_mock_sa_agent_result`, `_make_mock_failing_agent_result`) need the same change: no change to the constructor call (since `session_id` defaults to `None`), but the imports remain valid. This is a verify-only item â€” the existing code should work because `session_id` has a default value.
