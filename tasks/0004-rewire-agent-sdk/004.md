# 004: Rewrite AgentSession for ClaudeSDKClient

## Context

The `AgentSession` class is the core of the agent framework. It currently implements a custom streaming agentic loop using `AsyncAnthropic.messages.stream()` — it drives multi-turn conversations by manually detecting `tool_use` blocks, executing tools via `_execute_tool()`, feeding results back as `tool_result` messages, and continuing until `stop_reason == "end_turn"`. With the SDK switch, all of this loop management is handled by the Claude Code CLI process. The session becomes an observer: it sends a prompt, watches the message stream, emits events, and collects the result.

This is the most complex step in the task. The entire `session.py` file (~446 lines) is replaced, and the entire `test_agent_session.py` file (~1065 lines) is rewritten with a completely different mocking strategy.

Steps 002 and 003 have already set up the prerequisites — MCP tools, server factory, and role-based tool resolution. This step wires them together through the new `AgentSession` implementation.

## Reuse Opportunities

- **Retry logic structure — ~80% reuse**: The `run()` method's retry loop structure (attempt counter, exponential backoff, transient vs. non-transient branching, context overflow escalation, auth error detection) is preserved. The inner call changes from `self._run_once(prompt)` to the SDK client lifecycle. The constants `MAX_RETRIES = 3` and `RETRY_BASE_DELAY = 2.0` stay exactly as they are.

- **`_is_context_overflow()` — ~90% reuse**: The string matching logic for "prompt is too long", "context_length_exceeded", "maximum context length", "too many tokens" stays. May also need to check the `ResultMessage.result` text, since the SDK reports context overflow as an `is_error` result rather than an exception.

- **`_is_transient_error()` — reworked**: The function is rewritten to use SDK error types instead of Anthropic exception types. `CLIConnectionError` is transient. `MessageParseError` is not transient. The string-based 5xx and overload/rate-limit detection carries over from the current implementation.

- **`AgentResult` dataclass — extended**: The existing fields (`is_error`, `result_text`, `total_cost_usd`, `num_turns`, `full_output`) stay. A `session_id` field is added. The `full_output` field continues to capture all text from the session (now from `AssistantMessage` text blocks instead of streaming deltas).

- **Event emission pattern — adapted**: The concept of emitting `AgentTextChunk` events stays, but the emission point changes. Instead of emitting per token delta from the streaming API, events are emitted per text block within each `AssistantMessage` received from `receive_response()`. The `AgentTextChunk` dataclass and `EventEmitter` are unchanged.

- **Error classes — reused**: `AgentError` and `ContextOverflowError` stay as the public error types. Their constructor signatures may need minor adjustment (e.g., `ContextOverflowError` accepting optional `session_id`).

- **`_is_auth_error()` — merged into inline checks**: The standalone function can be replaced by inline string checks within the error handling block, matching the cc-runner pattern (which checks for "not found", "not installed", "auth", "login", "token" in the error string).

## Deliverables

### AgentResult dataclass update

The `AgentResult` dataclass is updated:

- **Add** a `session_id: str | None` field, defaulting to `None`. This captures the session identifier from the SDK's `ResultMessage` for diagnostics and potential future resume support.
- **Keep** all existing fields: `is_error`, `result_text`, `total_cost_usd`, `num_turns`, `full_output`.
- The `result_text` field is populated from `ResultMessage.result` (the SDK's final result text). If `ResultMessage.result` is `None` or empty, `result_text` falls back to `full_output` — this preserves the current behavior where `result_text` always contains useful content. Note: in the current implementation, `result_text` and `full_output` are set to the same value. After this change they may differ, but the fallback ensures `result_text` is never empty when there was actual text output.
- The `full_output` field is populated by concatenating all text from `AssistantMessage` text blocks throughout the session. This captures the complete text the agent produced across all turns.
- Cost comes from `ResultMessage.total_cost_usd` instead of token-based estimation. This is the actual cost reported by the CLI.

### Transient error detection rewrite

The `_is_transient_error()` function is rewritten for SDK error types:

- **`CLIConnectionError`**: Transient — the CLI process died or disconnected. Worth retrying.
- **`MessageParseError`**: Not transient — the SDK couldn't parse a message type. Retrying will hit the same parse failure.
- **String matching** (preserved from current): "500" + "internal server error" or "api_error" → transient. "502", "503", "529" → transient. "overloaded", "rate limit" → transient.
- **Removed**: All Anthropic SDK exception type checks (`RateLimitError`, `OverloadedError`, `AuthenticationError`), `status_code` attribute checks. These types no longer exist in the dependency tree.

The `_is_context_overflow()` function stays mostly unchanged — same string matching, but also checks for context overflow in `ResultMessage` error text (the SDK reports it as a result with `is_error=True`).

The standalone `_is_auth_error()` function is removed. Auth detection moves inline into the error handling block within `run()`, matching the cc-runner pattern: check for "not found", "not installed", "auth", "login", "token" in the error string.

### AgentSession class rewrite

The `AgentSession` class is completely rewritten:

**Constructor changes:**

- **Remove** the `client: AsyncAnthropic | None` parameter. There is no shared client.
- **Keep**: `config: AgentConfig`, `emitter: EventEmitter`, `context_id: str`, `agent_label: str`, `data_dir: Path | None`.
- The `data_dir` parameter stays and defaults to `Path.cwd() / "data"` as before. It is used when creating the MCP tool server.

**New `_build_options()` method:**

An internal method that constructs `ClaudeAgentOptions` for the SDK. It:

1. Resolves the `allowed_tools` list from the config's role (using `resolve_allowed_tools()` from step 003, or an empty list if role is `None`).
2. Resolves the MCP tool set from the role (using `resolve_tool_set()` from step 003).
3. If the role needs MCP file tools, creates the MCP server via `create_tools_server()` from step 002, passing `data_dir` and the tool set identifier. Otherwise, the MCP servers dict is empty.
4. Constructs `ClaudeAgentOptions` with:
   - `model` from the config
   - `system_prompt` from the config
   - `allowed_tools` from the resolved list
   - `mcp_servers` from the created server (if any)
   - `permission_mode` set to `"bypassPermissions"` — auto-approves all tool calls without interactive prompts, appropriate for automated agent runs. Tool access is controlled through `allowed_tools` and MCP server config, not runtime approval. The available modes are `Literal["default", "acceptEdits", "plan", "bypassPermissions"]`.
   - `cwd` set to the `data_dir` path (so the CLI operates within the data directory)

**Rewritten `run()` method:**

The method keeps the retry wrapper structure but replaces `_run_once()` with the SDK client lifecycle:

1. Calls `_build_options()` to get `ClaudeAgentOptions`.
2. Inside the retry loop, runs the SDK lifecycle:
   - `async with ClaudeSDKClient(options=options) as client:`
   - `await client.query(prompt)`
   - `async for message in client.receive_response():`
     - For `AssistantMessage` (detected via `type(message).__name__ == "AssistantMessage"`): iterates content blocks, extracts text from blocks with `hasattr(block, "text")`, appends to full output, emits `AgentTextChunk` events.
     - For `ResultMessage` (detected via `hasattr(message, "is_error")`): extracts `is_error`, `result`, `total_cost_usd`, `num_turns`, `session_id`. Checks for context overflow in error text.
3. After the message loop, constructs and returns `AgentResult` from the collected data.

The retry logic within `run()`:

- Catches `ContextOverflowError` and re-raises immediately (no retry).
- Catches other exceptions and classifies them:
  - Context overflow detected in error string → raises `ContextOverflowError`.
  - Auth error detected in error string → raises `AgentError` immediately.
  - Transient error and attempts remaining → waits with exponential backoff, continues.
  - Not transient or retries exhausted → raises `AgentError`.

**Removed methods:**

- `_run_once()` — the single-attempt method containing the Anthropic streaming loop. Replaced by the inline SDK lifecycle in `run()`.
- `_execute_tool()` — manual tool dispatch. The SDK handles tool execution via MCP.
