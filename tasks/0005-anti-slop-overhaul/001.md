# 001: Anti-slop preamble and prompt loading

## Context

The policy factory produces sanitized, committee-speak output because LLMs default to "helpful corporate communicator" mode and no prompt in the system counteracts this. A POC with vanilla Claude proved that a bias-acknowledge-then-override pattern — where the model names its specific sanitization tendencies for the topic at hand and commits to overriding them — produces genuinely sharp, analytical output.

An existing meditation preamble file (`sections/meditation.md`) asks the model to reflect on bias categories abstractly but never tells it to override its defaults. Worse, this file is never actually loaded in production — the `load_section()` mechanism exists in the loader but `build_agent_prompt()` is a trivial pass-through to `load_prompt()` without any section prepending.

This step creates the redesigned anti-slop preamble and wires it into all agent prompts by modifying the single choke point (`build_agent_prompt()`), so all 13 call sites across the codebase get the preamble automatically without any caller changes.

## Related Code

- `src/policy_factory/agent/prompts.py` — The `build_agent_prompt()` function. Currently a one-line pass-through to `load_prompt()`. This is the function to modify to auto-prepend the anti-slop section. Called from 13 sites across heartbeat orchestrator, cascade orchestrator, critic runner, synthesis runner, classifier, idea evaluator/generator, and seed router.
- `src/policy_factory/prompts/loader.py` — The `PromptLoader` class with `load_section()` and `load_sections()` methods already implemented. The `load_section()` method reads from `prompts/sections/{name}.md` with no variable substitution. The module-level `load_section()` convenience function delegates to the singleton loader.
- `src/policy_factory/prompts/sections/meditation.md` — The current meditation preamble (to be replaced). Generic 10-to-1 countdown reflecting on bias categories.
- `src/policy_factory/prompts/__init__.py` — Exports `load_section` and `load_sections` from the loader module. The docstring example references `load_section("meditation")` — update to reference the new section name.
- `tests/test_agent_prompts.py` — Existing tests for `build_agent_prompt()`. Several tests explicitly assert that meditation is NOT prepended and that the result equals exactly the template content. These tests must be rewritten to expect the anti-slop preamble IS prepended.

## Reuse Opportunities

- The `load_section()` mechanism in `PromptLoader` already handles loading section files from `prompts/sections/` with no variable substitution. This is exactly what's needed — `build_agent_prompt()` just needs to call it and prepend the result.
- The existing test fixture pattern in `test_agent_prompts.py` (creating a temp prompts directory, patching the singleton loader via `monkeypatch`) should be extended to also create a `sections/anti-slop.md` file in the temp directory so tests exercise the real prepending behavior.

## Deliverables

### 1. Replace `sections/meditation.md` with `sections/anti-slop.md`

Delete (or rename) the existing `sections/meditation.md` file. Create a new `sections/anti-slop.md` file implementing the bias-acknowledge-then-override pattern with four components:

**Bias acknowledgment:** Instruct the model to identify its specific training-induced sanitization tendencies for the topic it's about to analyze. This must be topic-aware, not a generic checklist — the model names what it would normally euphemize, soften, or avoid about this particular subject.

**Override commitment:** The model explicitly commits to counteracting each identified tendency in its output.

**Anti-slop directive:** Concrete prohibitions against euphemisms, corporatespeak, politician language, hedging, and filler. Include specific banned phrases (e.g., "stakeholder engagement," "multifaceted challenges," "this represents a significant development," "in the context of Finland's broader strategic landscape," "a concrete manifestation of") and examples contrasting bad vs. good output so the model understands the expected calibration. Include at least 3 example pairs to anchor the quality bar, such as:
- Bad: "This represents a significant development in Finland's strategic landscape" → Good: "Russia's naval exercises in the Baltic increased 40% this year — Finland's eastern border is now the most active NATO frontier"
- Bad: "There are multifaceted challenges related to demographic sustainability" → Good: "Finland's birth rate hit 1.32 in 2024 — the lowest in Nordic history. Immigration cannot close the gap at current integration rates"
- Bad: "Stakeholder engagement across multiple sectors will be critical for addressing emerging policy considerations" → Good: "The defense ministry needs €2B more annually but the finance ministry won't cut welfare. Something breaks."

The examples above are for calibration — the builder should write examples relevant to the preamble's voice and purpose, not copy these verbatim.

**Writing standard:** Direct, analytical, uncomfortable when the analysis demands it. The test is: if a sentence could appear in a government press release or EU white paper, it's wrong. The system's worldview is anchored to Finnish national interests and the model should be willing to produce output that would upset any political faction when the analysis demands it.

The preamble must be model-agnostic in wording — it will be prepended to prompts sent to both Claude and Gemini models. Focus on concrete instructions and examples rather than abstract philosophical framing. The preamble should not contain any `{variable}` placeholders since sections are loaded without variable substitution.

### 2. Modify `build_agent_prompt()` to auto-prepend the anti-slop preamble

Change `build_agent_prompt()` to load the anti-slop section via `load_section("anti-slop")` and prepend it to the loaded prompt content, separated by a double newline. The function signature remains unchanged — no callers need modification.

The import of `load_section` is available from `policy_factory.prompts` (already exported in `__init__.py`).

If the anti-slop section file is missing, the function should raise `FileNotFoundError` (the natural behavior of `load_section()`). The preamble is not optional — a missing section file is a deployment error.

### 3. Update `prompts/__init__.py` docstring

Update the docstring example that references `load_section("meditation")` to reference `load_section("anti-slop")` instead.

### 4. Update existing tests in `test_agent_prompts.py`

The existing test file has tests that explicitly assert meditation is NOT prepended and that results equal exactly the template content. These assertions must be inverted/updated:

- Tests should now verify that the anti-slop preamble IS prepended to all prompt outputs
- Tests should verify the preamble appears before the template content, separated by a double newline
- Tests should verify template variable substitution still works correctly with the preamble prepended
- Tests should verify that the preamble itself does not undergo variable substitution (it has no placeholders)
- Tests should verify `FileNotFoundError` is raised if the anti-slop section file is missing
- Tests should verify `FileNotFoundError` is raised if the template file is missing (existing behavior)

The temp prompts fixture must be extended to create a `sections/` subdirectory with a test `anti-slop.md` file.

The "real prompts" integration tests (`TestBuildAgentPromptWithRealPrompts`) should be updated to verify that real prompt files load successfully WITH the anti-slop preamble prepended.
