# 003: SA seed and generator prompts

## Context

Six prompt files need rewriting in this step: the SA seed prompt and all five generator prompts (values, situational-awareness, strategic, tactical, policies). All six reference "cross-party tech policy analysis system" or "tech policy" and contain tech-centric topic lists or evaluation criteria.

The SA seed and generators are the workhorses of the system — they produce the actual policy content. Every one of them currently produces sanitized, scope-limited output because their instructions frame everything as "technology policy." Broadening to general policy and reinforcing anti-slop writing standards in each prompt's own instructions ensures the content quality matches the preamble's intent.

The anti-slop preamble (step 001) will be auto-prepended to all of these. However, each prompt should also reinforce the writing standard in its own instructions section — the preamble sets the mindset, and the per-prompt instructions specify what good output looks like for that specific agent role.

## Related Code

- `src/policy_factory/prompts/seed/seed.md` — SA seed prompt. Uses template variables `{current_date}` and `{values_content}`. Called from `server/routers/seed.py` which passes these variables. Uses `web_search` tool to research current information. Lists 10 tech-centric topics (EU regulatory landscape, AI and emerging technology, cybersecurity, digital infrastructure, etc.).

- `src/policy_factory/prompts/generators/values.md` — Values generator prompt. Uses template variables `{layer_content}`, `{feedback_memos}`, `{cross_layer_context}`. Uses file tools (`list_files`, `read_file`, `write_file`, `delete_file`) to manage value files. Lists 10 tech-centric evaluation criteria. Must be updated to align with the tension-pair values model from step 002.

- `src/policy_factory/prompts/generators/situational-awareness.md` — SA generator prompt. Uses template variables `{upstream_content}`, `{layer_content}`, `{feedback_memos}`, `{cross_layer_context}`. Uses file tools. Lists 10 tech-centric assessment areas. Needs the strongest anti-slop reinforcement — this is the layer that should produce uncomfortable geopolitical analysis.

- `src/policy_factory/prompts/generators/strategic.md` — Strategic objectives generator. Uses template variables `{upstream_content}`, `{layer_content}`, `{feedback_memos}`, `{cross_layer_context}`. Uses file tools. References "tech policy analysis system." Good structural framework (5-15 year horizon, feedback memos) but tech-scoped.

- `src/policy_factory/prompts/generators/tactical.md` — Tactical objectives generator. Same template variables and tool pattern as strategic. References "tech policy analysis system." Good structural framework (1-5 year horizon, concrete and actionable) but tech-scoped.

- `src/policy_factory/prompts/generators/policies.md` — Policies generator. Same template variables and tool pattern. References "tech policy analysis system." Has a solid output format section (Summary, Rationale, Implementation, Cost Estimate, Political Feasibility, International Context, Risks) that should be preserved and broadened.

- `src/policy_factory/cascade/orchestrator.py` — Contains `_SLUG_TO_PROMPT` mapping and `_run_generation_step()` which calls `build_agent_prompt("generators", prompt_name, ...)`. Also has `_gather_generation_context()` for assembling layer context. This code is not modified — only the prompts change.

- `src/policy_factory/server/routers/seed.py` — Contains the SA seed endpoint that calls `build_agent_prompt("seed", "seed", current_date=..., values_content=...)`. This code is not modified.

## Reuse Opportunities

- All five generator prompts share the same structural skeleton: context section (five-layer model description), template variable injection sections (upstream content, layer content, feedback memos, cross-layer context), file tool instructions, output format, and "Important" closing directives. This shared structure should be preserved — only the content-specific parts change.
- The policies generator has the most detailed output format section (7 structured subsections). This pattern of structured output with explicit subsection headings is good and should be preserved, with scope broadened from tech to general policy.
- The SA generator's existing instruction to "Include uncomfortable truths. If Finland is falling behind in an area, say so" is the right spirit and should be significantly strengthened in the rewrite.
- The strategic generator's instruction about feedback memos for layers below is a good cross-layer communication pattern that should be preserved.

## Deliverables

### 1. Rewrite `src/policy_factory/prompts/seed/seed.md`

Replace all "tech policy" framing with general policy scope. The current 10 tech-centric topics must be replaced with general policy domains covering the full breadth of Finnish policy:

- **Geopolitical and security** — NATO membership, EU integration, Russia, Arctic, US/China dynamics, nuclear umbrella reliability, Nordic defense cooperation
- **Immigration and demographics** — Population aging, labor immigration, refugee policy, integration outcomes, ethnic composition trends
- **Economics and taxation** — Competitiveness, industrial structure, public debt, tax burden, trade dependencies, innovation capacity
- **Energy and environment** — Energy security, nuclear policy, green transition, resource dependencies, climate commitments
- **Social policy and welfare** — Nordic model sustainability, healthcare, pension system, inequality trends, work incentives
- **Defense and military** — Capabilities, conscription, defense spending, NATO interoperability, military-industrial base
- **EU relations** — Integration depth, sovereignty trade-offs, regulatory influence, net contributor status
- **Trade and economic dependencies** — Supply chain vulnerabilities, key trading partners, sanctions exposure
- **Drugs, public health, and social order** — Drug policy, mental health, public safety, migration-related tensions
- **Education and workforce** — Education system, research capacity, brain drain/gain, skill gaps

The prompt must instruct the agent to produce genuine geopolitical analysis, not surface-level summaries. The SA seed creates the foundational factual picture that all higher layers build upon — if this layer is Wikipedia-grade, everything above it will be too.

Preserve the template variables `{current_date}` and `{values_content}`. Preserve the instruction to use `web_search` for current information and `write_file` for output. Preserve the output format structure (one markdown file per topic with YAML frontmatter).

### 2. Rewrite `src/policy_factory/prompts/generators/values.md`

Update to align with the tension-pair values model established in step 002. The current prompt asks the generator to evaluate values against 10 tech-centric criteria ("National security and sovereignty", "Technological competitiveness and digital infrastructure", etc.). Replace with instructions to:

- Evaluate whether the current tension-pairs remain relevant and genuinely controversial
- Identify tensions that have shifted or resolved and should be updated or removed
- Identify new tensions that have emerged from cross-layer feedback or changing circumstances
- Maintain the tension-pair structure: each value as a genuine dilemma with analysis of Finland's actual position
- Incorporate feedback memos from higher layers about values that are unclear, missing, or producing bad policy outcomes

Replace "cross-party tech policy analysis system" with general policy framing. Reinforce that values must be controversial enough to guide real policy decisions — if a value has become consensus, it should be removed.

Preserve template variables `{layer_content}`, `{feedback_memos}`, `{cross_layer_context}`. Preserve file tool instructions.

### 3. Rewrite `src/policy_factory/prompts/generators/situational-awareness.md`

This prompt needs the strongest anti-slop reinforcement. Replace the 10 tech-centric assessment areas with the same general policy domains listed for the SA seed above. Additionally:

- Instruct the agent to produce intelligence-briefing quality analysis, not encyclopedia summaries
- The standard is: would a Finnish policymaker learn something they didn't already know? If not, the analysis isn't deep enough
- Uncomfortable geopolitical analysis is explicitly required — can the US nuclear umbrella actually be trusted? Is the EU moving toward a transfer union? What are the realistic scenarios for Russia post-war?
- The agent should distinguish between: publicly known facts (minimize repetition), expert-consensus assessments (state clearly), genuinely uncertain/contested situations (analyze competing views), and the agent's own analytical conclusions (state as such)

Replace "cross-party tech policy analysis system" with general policy framing. Preserve template variables and file tool instructions.

### 4. Rewrite `src/policy_factory/prompts/generators/strategic.md`

Replace "cross-party tech policy analysis system" with general policy framing. The structural framework is sound (5-15 year horizon, grounded in values AND reality, feedback memos for layers below). Changes:

- Broaden evaluation from tech-specific strategic concerns to all policy domains
- Reinforce that strategic objectives should address genuine trade-offs — if a strategic objective doesn't require sacrificing something, it's not honest
- Strategic objectives should be willing to state uncomfortable positions (e.g., "Accept that demographic decline is irreversible and plan for managed population shrinkage" if the analysis supports it)

Preserve template variables and file tool instructions.

### 5. Rewrite `src/policy_factory/prompts/generators/tactical.md`

Replace "cross-party tech policy analysis system" with general policy framing. The structural framework is sound (1-5 year horizon, concrete and actionable, resource-aware). Changes:

- Broaden from tech-specific tactical concerns to all policy domains
- Reinforce that tactical objectives should name specific institutional actors and resource requirements
- Be honest about political difficulty — if a tactical objective is good policy but politically toxic, say so

Preserve template variables and file tool instructions.

### 6. Rewrite `src/policy_factory/prompts/generators/policies.md`

Replace "cross-party tech policy analysis system" with general policy framing. This prompt has the best output structure of all generators (Summary, Rationale, Implementation, Cost Estimate, Political Feasibility, International Context, Risks). Preserve this structure. Changes:

- Broaden the policy scope from tech to all domains
- Strengthen the "be honest about political difficulty" instruction — policies should explicitly name which political factions would oppose them and why
- Policies should include an honest assessment of whether they would actually work, including counterarguments and failure modes

Preserve template variables and file tool instructions.
