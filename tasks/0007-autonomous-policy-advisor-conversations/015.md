# 015: Item page sidebar integration

## Context

The ItemDetailPage needs to integrate the ConversationSidebar component. This involves adding a toggle button to open the conversation sidebar, adjusting the page layout so content shifts left when the sidebar is open, and wiring up live content refresh when the AI edits the item during conversation.

## Related Code

Existing page structure:

- `ui/src/pages/ItemDetailPage.tsx` — Current item view/edit page with PageWrapper (800px max-width centered), section-based layout, data fetching via slug and filename params, edit mode state management
- `ui/src/pages/ItemDetailPage.styles.ts` — Co-located styles with layer-specific coloring via getLayerColors helper, responsive design patterns

Components to integrate:

- `ui/src/components/organisms/ConversationSidebar.tsx` — (from step 014) the sidebar component with isOpen, onClose, layerSlug, filename props

Store integration:

- `ui/src/stores/conversationStore.ts` — (from step 012) for initializing conversation context
- `ui/src/stores/layerStore.ts` — refresh() method that gets called on file edit events

Patterns for toggle buttons:

- `ui/src/components/atoms/IconButton.ts` — Icon button for toggle
- `lucide-react` — Icons used throughout the app

## Reuse Opportunities

- Use existing IconButton atom for the conversation toggle
- Follow the existing page header action button pattern (Edit, Delete buttons) for toggle placement
- Use layerStore subscription pattern for detecting when to refresh item data
- Follow the existing PageWrapper/Section layout patterns

## Deliverables

### Toggle Button

Add a conversation toggle button to the page header:
- Place in the header actions area alongside Edit/Delete buttons
- Use a chat/message icon (e.g., MessageSquare from lucide-react)
- Toggle opens/closes the sidebar
- Visual indicator when sidebar is open (active state styling)

### Sidebar State

Add local state for sidebar visibility:
- isConversationOpen boolean state
- Toggle function for the button
- Initialize conversationStore context with layerSlug and filename when sidebar opens

### Layout Adjustment

Modify the page layout to accommodate the open sidebar:
- When sidebar is closed, page content centers normally (current behavior)
- When sidebar is open, content area shifts or narrows to make room for sidebar
- Options: shift content left, reduce max-width, or use a two-column layout
- Transition smoothly when opening/closing
- Ensure content remains readable at narrower width

### Content Refresh

Wire up live content refresh when AI edits the current item:
- Subscribe to layerStore or listen for file edit events
- When a conversation_file_edit event affects the current item (matching slug and filename), refetch item data
- Update the view with fresh content
- Handle case where item is being edited (warn user or merge changes)

### Edit Mode Interaction

Handle interaction between edit mode and conversation:
- If user is in edit mode with unsaved changes when AI edits the same item, this creates a conflict
- When AI edits the current item while user has unsaved changes:
  - Display an inline banner/toast notification at the top of the content area
  - Banner text: "This item was modified by the AI. Your unsaved changes may conflict."
  - Banner includes a "Refresh" button to load the AI's version (discarding user edits)
  - Banner includes a "Keep editing" button to continue with user's version
- The conversation sidebar remains usable regardless of edit mode

### Sidebar Props

Pass correct props to ConversationSidebar:
- isOpen: from local sidebar state
- onClose: callback to close sidebar
- layerSlug: from URL params
- filename: from URL params

- [x] Clicking toggle again (or close button) closes sidebar

- [x] Layout transition is smooth/animated
- [x] Content remains readable when sidebar is open
- [x] ConversationSidebar receives correct layerSlug and filename
- [x] Item content refreshes when AI edits the current item
- [x] Refresh happens automatically without manual user action

- [x] Banner offers "Refresh" and "Keep editing" options
- [x] Toggle button shows active state when sidebar is open
