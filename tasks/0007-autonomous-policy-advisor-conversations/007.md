# 007: Conversation agent config

## Context

The conversation feature requires a new agent role with full file tool permissions. The conversation agent needs to read, write, and delete files across the entire data directory during discussions. It must use the Claude CLI backend (not Gemini) because it requires tool-using capabilities.

This step adds the "conversation" role to the agent configuration system, enabling the conversation runner to create properly configured AgentSession instances.

## Related Code

Existing agent configuration:

- `src/policy_factory/agent/config.py` — AgentRole literal type, _DEFAULT_MODELS mapping, _ENV_VAR_MAP for environment overrides, _ALLOWED_TOOLS_BY_ROLE for SDK permissions, _TOOL_SET_BY_ROLE for MCP tool sets, resolve_model() and resolve_tool_set() functions
- `src/policy_factory/agent/tools.py` — TOOL_SET_FULL, TOOL_SET_READ_ONLY, TOOL_SET_NONE constants defining available file operations
- `src/policy_factory/agent/session.py` — AgentSession that uses config to build agent options

Example roles to follow:

- "generator" role — uses Claude CLI default model (None), has TOOL_SET_FULL, has MCP_SERVER_REF in allowed tools
- "critic" role — uses Claude CLI default model, has TOOL_SET_READ_ONLY
- "seed" role — uses Claude CLI default model, has TOOL_SET_FULL for initial content generation

## Reuse Opportunities

- Follow the exact pattern of the "generator" role since conversation needs the same capabilities: Claude CLI for tool support, full file permissions, MCP server access
- The resolve_model() and resolve_tool_set() functions already handle the lookup logic — just need to add entries to the mappings
- No new functions needed, just dictionary entries

## Deliverables

### AgentRole Update

Add "conversation" to the AgentRole literal type union.

### Model Configuration

Add "conversation" to _DEFAULT_MODELS mapping:
- Set to None to use Claude CLI default model (Opus) since conversation needs tool-using capability

### Environment Variable

Add "conversation" to _ENV_VAR_MAP:
- Map to "POLICY_FACTORY_MODEL_CONVERSATION" for runtime override capability

### Tool Permissions

Add "conversation" to _ALLOWED_TOOLS_BY_ROLE:
- Grant MCP_SERVER_REF to enable file tools

Add "conversation" to _TOOL_SET_BY_ROLE:
- Grant TOOL_SET_FULL (list_files, read_file, write_file, delete_file)

### Search Configuration

Add "conversation" to _USE_SEARCH_BY_ROLE if that mapping exists:
- Set to False (conversation doesn't need web search grounding by default)

## Acceptance Criteria

- [x] "conversation" is a valid AgentRole literal value
- [x] resolve_model("conversation") returns None (uses CLI default)
- [x] Setting POLICY_FACTORY_MODEL_CONVERSATION environment variable overrides the model
- [x] resolve_allowed_tools("conversation") includes MCP_SERVER_REF
- [x] resolve_tool_set("conversation") returns TOOL_SET_FULL
- [x] AgentSession created with role="conversation" has access to all file tools
- [x] AgentSession with role="conversation" can write files via write_file tool
- [x] AgentSession with role="conversation" can delete files via delete_file tool
