# 003: Philosophy layer UI

## Context

The philosophy layer needs visual identity in the frontend to appear alongside existing layers. This includes theme colors for badges and layer indicators, i18n translation strings for the display name, and layer constant mappings for dynamic rendering.

This step focuses only on the styling and constants infrastructure. The philosophy layer will automatically appear in layer lists and navigation once these constants are in place, since existing components use slug-based dynamic lookups.

## Related Code

Existing files that define layer visual identity:

- `ui/src/styles/theme.ts` — Theme interface with layers object containing LayerColors (primary, bg, text) for each layer slug. Defines colors in both darkTheme and lightTheme.
- `ui/src/lib/layerConstants.ts` — LAYER_NAME_KEYS mapping layer slugs to i18n translation keys
- `ui/src/i18n/locales/en.ts` — Translation strings including layer display names in stackOverview section

Components that consume layer styling dynamically:

- `ui/src/pages/StackOverviewPage.tsx` — Uses theme.colors.layers[layerSlug] for layer visualization
- `ui/src/components/atoms/Badge.ts` — Badge component that can be styled with layer colors
- Layer detail and item detail pages use layer colors for headers and accents

## Reuse Opportunities

- The existing LayerColors interface (primary, bg, text) can be directly extended with a new entry for philosophy
- The LAYER_NAME_KEYS pattern is a simple key-value addition
- The i18n structure under stackOverview already has a consistent pattern (layerValues, layerSituationalAwareness, etc.)
- Components already use dynamic slug-based lookups, so no component changes are needed — just add the constants

## Deliverables

### Theme Colors

Add philosophy layer colors to the Theme interface and both theme implementations:

- Update the Theme interface's layers object to include "philosophy" with LayerColors type
- Add philosophy colors to darkTheme.colors.layers with appropriate primary, bg, and text values
- Add philosophy colors to lightTheme.colors.layers with appropriate primary, bg, and text values
- Choose a color that is visually distinct from existing layer colors (values=purple, situational-awareness=cyan, strategic=blue, tactical=yellow/orange, policies=green). A deep rose/pink or warm red tone would provide good differentiation as the foundational layer.

### Layer Constants

Add philosophy to LAYER_NAME_KEYS mapping:

- Add entry mapping "philosophy" slug to its i18n key path

### i18n Strings

Add philosophy display name to English translations:

- Add layerPhilosophy entry under stackOverview section with display name "Philosophy"
