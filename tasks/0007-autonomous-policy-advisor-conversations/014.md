# 014: Conversation sidebar component

## Context

The conversation sidebar is the primary UI for the conversation feature. It's a right-aligned sliding panel that appears alongside item or layer content, containing the conversation history selector, message list with streaming support, input textarea, and pending cascade notification banner. This is a significant UI component that needs to handle real-time streaming, user input, and visual feedback for AI activity.

## Deliverables

### Component Files

Create `ui/src/components/organisms/ConversationSidebar.tsx` and `ui/src/components/organisms/ConversationSidebar.styles.ts`.

### Sidebar Container

The sidebar should:
- Be fixed-positioned on the right side of the viewport
- Have a consistent width (400-500px)
- Slide in/out with animation when opened/closed
- Use appropriate z-index from theme (above content, below modals)
- Have a header with title and close button

### Conversation Selector

When multiple conversations exist for the item/layer:
- Display a dropdown or list to switch between conversations
- Show conversation previews (last message, timestamp)
- Include a "New Conversation" option
- When no conversations exist, show a prompt to start one

### Message List

Display conversation messages:
- Scroll container for messages
- Visual distinction between user messages and assistant messages (alignment, background color, or border)
- Display message timestamp
- For assistant messages, show files_edited indicator if any files were modified
- Auto-scroll to bottom when new messages arrive
- Handle long messages gracefully (scrollable or expandable)

### Streaming Display

During AI response generation:
- Show streaming indicator (typing animation or similar)
- Display streamed text as it accumulates in real-time
- Keep the streaming message visually distinct until complete
- Auto-scroll as streaming text grows

### Message Input

User message composition:
- Multi-line textarea for message input
- Send button (enabled when input is non-empty and not streaming)
- Keyboard shortcut: Ctrl+Enter or Cmd+Enter to send
- Disable input while AI is responding (streaming state)
- Clear input after successful send

### Pending Cascade Banner

When pendingCascade is set in the store:
- Display a prominent banner at top or bottom of sidebar
- Show the affected starting layer
- "Trigger Cascade" button that calls the trigger-pending API
- "Dismiss" button that clears the pending cascade without triggering
- Banner should be visually distinct (warning/info color)

### Error States

Handle error conditions:
- Display error message if conversation load fails
- Display error indicator on failed message sends
- Show error state for conversation_turn_error events
- Include retry option where appropriate

### Loading States

Handle loading conditions:
- Show loading indicator when fetching conversations
- Show loading indicator when fetching messages
- Skeleton or placeholder during initial load

### Store Integration

Connect to conversationStore:
- Subscribe to conversations, activeConversation, messages
- Subscribe to isStreaming, streamingText for real-time display
- Subscribe to pendingCascade for banner display
- Call store actions for CRUD operations and message sending

### Props Interface

The component should accept:
- isOpen: boolean controlling visibility
- onClose: callback when user closes sidebar
- layerSlug: which layer context
- filename: optional, which item context (null for layer-level)
