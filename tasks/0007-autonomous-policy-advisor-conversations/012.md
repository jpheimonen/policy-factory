# 012: Conversation store (frontend)

## Context

The conversation feature needs a Zustand store to manage client-side state for conversations. This includes tracking the active conversation, message list, streaming state during AI responses, and the pending cascade notification flag. The store integrates with both REST API calls (for CRUD operations) and WebSocket events (for real-time streaming).

## Related Code

Existing Zustand stores to follow:

- `ui/src/stores/cascadeStore.ts` — Example of handling streaming text chunks (agent_text_chunk events), buffering streamed content, and managing run lifecycle state
- `ui/src/stores/activityStore.ts` — Example of hybrid REST + WebSocket pattern, fetchHistory for initial load, addEvent for real-time updates
- `ui/src/stores/ideaStore.ts` — Example of CRUD actions with loading/error states, optimistic updates
- `ui/src/stores/layerStore.ts` — Example of detail view state (detailItems, detailSummary) alongside list state

Event dispatch integration:

- `ui/src/hooks/useEventDispatch.ts` — Central dispatcher that routes events to store handlers

API client:

- `ui/src/lib/apiClient.ts` — apiRequest utility for REST calls

## Deliverables

### New Store File

Create `ui/src/stores/conversationStore.ts` with the conversation Zustand store.

### State Shape

Define state to track:

**Conversation List State**
- conversations: list of conversation summaries for the current item/layer
- conversationsLoading: boolean for list loading state
- conversationsError: string or null for list errors

**Active Conversation State**
- activeConversationId: string or null
- activeConversation: conversation detail object or null
- messages: list of messages for the active conversation
- messagesLoading: boolean for messages loading state
- messagesError: string or null for messages errors

**Streaming State**
- isStreaming: boolean indicating if AI is currently responding
- streamingText: string buffer accumulating streamed text chunks
- streamingMessageId: string or null for the message being streamed

**Pending Cascade State**
- pendingCascade: object with starting_layer or null
- Shows when conversation edits triggered a cascade notification

**Context State**
- currentLayerSlug: string or null (what layer/item we're viewing conversations for)
- currentFilename: string or null (null for layer-level)

### REST Actions

**fetchConversations(layerSlug, filename?)**
- Fetches conversations for an item or layer
- Sets conversationsLoading during fetch
- Updates conversations list on success
- Sets conversationsError on failure

**fetchConversation(conversationId)**
- Fetches a single conversation with its messages
- Sets messagesLoading during fetch
- Updates activeConversation and messages on success
- Sets messagesError on failure

**createConversation(layerSlug, filename?)**
- Creates a new conversation via POST
- Adds the new conversation to the list
- Sets it as active
- Returns the new conversation ID

**sendMessage(conversationId, content)**
- Sends a user message via POST
- Adds the user message to messages list immediately (optimistic)
- Sets isStreaming to true
- Initializes empty streamingText buffer
- The actual AI response arrives via WebSocket events

**deleteConversation(conversationId)**
- Deletes a conversation via DELETE
- Removes from conversations list
- Clears active state if deleted conversation was active

### WebSocket Event Handlers

**handleConversationStarted(event)**
- Confirms streaming is starting
- May update streamingMessageId if provided

**handleConversationTextChunk(event)**
- Appends event.text to streamingText buffer
- Only processes if event.conversation_id matches activeConversationId

**handleConversationFileEdit(event)**
- Could track file edits for display in UI
- May trigger parent component to refresh item content

**handleConversationTurnComplete(event)**
- Sets isStreaming to false
- Creates final assistant message from streamingText
- Adds to messages list
- Clears streamingText buffer
- Updates files_edited on the message if provided

**handleConversationTurnError(event)**
- Sets isStreaming to false
- Creates error message or updates existing streaming message with error state
- Clears streamingText buffer

**handleConversationCascadePending(event)**
- Sets pendingCascade with starting_layer from event

### Utility Actions

**setActiveConversation(conversationId)**
- Sets activeConversationId
- Triggers fetchConversation if not already loaded

**clearActiveConversation()**
- Clears active state, messages, streaming state

**clearPendingCascade()**
- Clears pendingCascade (after user triggers or dismisses)

**setContext(layerSlug, filename?)**
- Sets currentLayerSlug and currentFilename
- Used when opening sidebar on a different item/layer

### Selectors

Components will select specific slices:
- conversations list for conversation picker
- messages for message display
- isStreaming and streamingText for streaming indicator
- pendingCascade for cascade notification banner
