# 011: Conversation API endpoints

## Context

The conversation feature needs REST API endpoints for creating conversations, listing conversations for an item or layer, retrieving conversation details with messages, sending messages (which triggers the agent), and deleting conversations. The message-sending endpoint is unique in that it triggers an async agent process and streams the response via WebSocket events rather than returning the full response synchronously.

## Deliverables

### New Router File

Create `src/policy_factory/server/routers/conversations.py` with the conversation API endpoints.

### Router Registration

Register the new router in app.py with prefix `/api/conversations`.

### Pydantic Models

Define request and response models:

**CreateConversationRequest**
- layer_slug: required string
- filename: optional string (null for layer-level conversations)

**ConversationResponse**
- id: string
- layer_slug: string
- filename: optional string
- created_at: ISO8601 string
- last_active_at: ISO8601 string

**ConversationListResponse**
- conversations: list of ConversationResponse

**MessageResponse**
- id: string
- role: string ("user" or "assistant")
- content: string
- created_at: ISO8601 string
- files_edited: optional list of strings

**ConversationDetailResponse**
- conversation: ConversationResponse
- messages: list of MessageResponse

**SendMessageRequest**
- content: string (the user's message text)

**SendMessageResponse**
- message_id: string (the user message ID)
- Note: assistant response streams via WebSocket events

### Endpoints

**POST /api/conversations**
- Creates a new conversation for an item or layer
- Requires authentication
- Request body: CreateConversationRequest
- Response: ConversationResponse with 201 status
- Validates that layer_slug is a valid layer
- If filename provided, validates the item exists

**GET /api/conversations**
- Lists conversations filtered by layer_slug and optional filename
- Requires authentication
- Query params: layer_slug (required), filename (optional)
- Response: ConversationListResponse
- If filename is provided, returns item-level conversations
- If filename is omitted, returns layer-level conversations (where filename is null)

**GET /api/conversations/{conversation_id}**
- Retrieves a conversation with all its messages
- Requires authentication
- Path param: conversation_id
- Response: ConversationDetailResponse
- Returns 404 if conversation not found

**POST /api/conversations/{conversation_id}/messages**
- Sends a user message and triggers the AI response
- Requires authentication
- Path param: conversation_id
- Request body: SendMessageRequest
- Response: SendMessageResponse with 202 status (accepted, processing)
- Stores the user message immediately
- Launches the conversation runner as a background task
- AI response streams via WebSocket events (ConversationTextChunk, etc.)
- Returns 404 if conversation not found

**DELETE /api/conversations/{conversation_id}**
- Deletes a conversation and all its messages
- Requires authentication
- Path param: conversation_id
- Response: 204 No Content
- Returns 404 if conversation not found

### Background Task Integration

The POST /messages endpoint launches the conversation runner asynchronously:
- Import run_conversation_turn from conversation/runner.py (step 009)
- Create asyncio task calling run_conversation_turn(conversation_id, content, store, emitter, data_dir)
- Return immediately with the user message ID
- The runner handles streaming events and storing the assistant message

### Error Handling

- 400 Bad Request: invalid layer_slug, invalid filename format
- 404 Not Found: conversation not found, item not found (when creating)
- 401 Unauthorized: missing or invalid authentication
