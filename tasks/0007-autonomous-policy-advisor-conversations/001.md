# 001: Philosophy layer definition

## Context

The philosophy layer is a new foundational layer that sits below values in the cascade hierarchy. It defines the system's reasoning axioms — epistemological commitments, normative axioms, and philosophical tradition identification. All other layers build upon this foundation.

Adding a new layer requires updates across multiple files that have hardcoded layer references. This step handles the backend data model and cascade integration only; UI changes and prompts are handled in subsequent steps.

## Related Code

Existing files that define or reference layers:

- `src/policy_factory/data/layers.py` — LAYERS list with LayerInfo dataclass, position numbering, and helper functions (get_layer, validate_layer_slug, layers utilities)
- `src/policy_factory/cascade/orchestrator.py` — _SLUG_TO_PROMPT mapping that connects layer slugs to prompt template filenames
- `src/policy_factory/ideas/helpers.py` — _SLUG_TO_VAR mapping that connects layer slugs to template variable names for idea evaluation prompts
- `src/policy_factory/server/routers/seed.py` — Seeding endpoints for each layer (explicit endpoints per layer)
- `src/policy_factory/agent/config.py` — Agent roles including layer-specific seed roles

## Reuse Opportunities

- The existing LayerInfo dataclass and LAYERS list pattern can be directly extended with a new entry
- The _SLUG_TO_PROMPT and _SLUG_TO_VAR dictionaries follow simple key-value patterns that just need a new entry
- The seeding endpoint pattern in seed.py can be copied for the philosophy seed endpoint
- The agent role pattern for layer-specific seeders (e.g., "values-seed") can be followed for "philosophy-seed"

## Deliverables

### Layer Definition
- Add philosophy layer to LAYERS list at position 1 (bottom of the stack)
- Renumber all existing layers: values becomes position 2, situational-awareness becomes position 3, etc.
- Philosophy layer uses slug "philosophy" and display name "Philosophy"

### Cascade Integration
- Add philosophy to _SLUG_TO_PROMPT mapping in orchestrator.py (maps to "philosophy" prompt template)
- The layers_from and layers_below utilities automatically adapt via the LAYERS list — no changes needed

### Idea Pipeline
- Add philosophy to _SLUG_TO_VAR mapping in helpers.py (maps to "philosophy_summary" variable name)

### Seeding Infrastructure
- Add philosophy-seed agent role to agent config with appropriate model and tool permissions
- Add POST /api/seed/philosophy endpoint following the pattern of other seed endpoints

### Foundational Layers Constant

Define FOUNDATIONAL_LAYERS constant in layers.py:
- A set containing {"philosophy", "values", "situational-awareness"}
- These are the layers (positions 1-3) that trigger cascade pending when edited via conversation
- Export this constant for use by the conversation runner

### Filesystem
- Create data/philosophy/ directory
- Create data/philosophy/README.md with placeholder narrative summary
