# 018: E2E tests

## Context

The conversation feature, philosophy layer UI, and streaming display all require end-to-end browser tests using Playwright. E2E tests verify the full user experience from clicking UI elements through real API calls to visible results, ensuring all components work together correctly.

## Related Code

Existing E2E test patterns:

- `ui/e2e/item-detail.spec.ts` — Pattern for testing item pages with API setup, navigation, and UI assertions
- `ui/e2e/layer-detail.spec.ts` — Pattern for testing layer pages
- `ui/e2e/cascade-viewer.spec.ts` — Pattern for testing streaming/real-time updates
- `ui/e2e/helpers.ts` — setupAndLogin, getAdminToken utilities
- `ui/e2e/global-setup.ts` — Database backup/restore pattern
- `ui/playwright.config.ts` — Test configuration, base URL, timeouts

New UI components to test (from previous steps):

- `ui/src/components/organisms/ConversationSidebar.tsx` — Sidebar component (step 014)
- `ui/src/pages/ItemDetailPage.tsx` — With sidebar integration (step 015)
- `ui/src/pages/LayerDetailPage.tsx` — With sidebar integration (step 016)

## Reuse Opportunities

- Follow the exact test structure pattern from existing spec files
- Reuse setupAndLogin and getAdminToken helpers
- Use page.request for API setup (creating test conversations, items)
- Follow the locator patterns: getByRole, getByText, getByLabel
- Use waitFor and expect with timeouts for async operations
- Follow the API seeding pattern from item-detail.spec.ts

Create multiple focused E2E test files in `ui/e2e/`:

### Conversation Sidebar Tests

`ui/e2e/conversation-sidebar.spec.ts`:

**Toggle and Basic Interaction**
- Conversation toggle button appears on item detail page
- Clicking toggle opens the right sidebar panel
- Sidebar animates in from the right
- Clicking toggle again closes the sidebar
- Clicking close button (X) closes the sidebar
- Pressing Escape key closes the sidebar
- Page layout adjusts when sidebar opens (content narrows)
- Layout returns to normal when sidebar closes

**Conversation Creation**
- Empty state shows "Start new conversation" prompt when no conversations exist
- Clicking "New Conversation" creates a new conversation
- New conversation shows empty message list
- Conversation appears in conversation selector dropdown

**Message Sending**
- Typing in textarea enables send button
- Empty textarea disables send button
- Clicking send creates user message bubble
- Ctrl+Enter keyboard shortcut sends message
- User message appears immediately after send (optimistic)
- Input clears after successful send
- Input is disabled while waiting for AI response

**Conversation Persistence**
- Closing and reopening sidebar shows same conversation
- Messages persist across page refreshes
- Multiple conversations for same item appear in selector
- Switching conversations loads correct message history

### Layer-Level Conversation Tests

Additional tests in conversation-sidebar.spec.ts:

**Layer Page Integration**

- Layer conversation sidebar functions identically to item sidebar
- Sidebar header shows layer name (not item name)
- Creating layer-level conversation works
- Messages send and display correctly

### Streaming Display Tests

`ui/e2e/conversation-streaming.spec.ts`:

**Real-Time Streaming**
- Assistant response streams incrementally (not all at once)
- Streaming text appears in real-time character by character
- Streaming indicator visible during AI response
- User can scroll the message list while streaming continues
- Auto-scroll keeps streaming message visible
- Final message content matches complete streamed text
- Streaming state clears when response completes

**Note:** These tests may require mocking the backend or using a test mode that streams predictable content with delays.

### Test Helpers

Add to `ui/e2e/helpers.ts`:

**createTestConversation(page, layerSlug, filename?)**
- Creates a conversation via API for test setup
- Returns conversation_id for further operations

**sendTestMessage(page, conversationId, content)**
- Sends a message via API for setup
- Useful for pre-populating conversation history

**waitForStreamingComplete(page)**
- Waits for streaming indicator to disappear
- Ensures AI response is fully received before assertions

### Test Data Setup

Tests should create their own test data via API:
- Create test items in philosophy layer for philosophy tests
- Create test conversations for retrieval tests
- Use unique filenames/IDs to avoid conflicts between tests
