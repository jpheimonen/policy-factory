# 006: Conversation events

## Context

The conversation feature needs real-time event streaming to power the UI. Events notify the frontend when a conversation turn starts, when response text chunks arrive for streaming display, when the agent edits files, when a turn completes, and when foundational layer edits queue a pending cascade.

The event system follows a well-established pattern: backend dataclass events are emitted via EventEmitter, persisted to SQLite with a db_id, broadcast via WebSocket, and handled by type-safe frontend interfaces.

## Related Code

Backend event definitions and emission:

- `src/policy_factory/events.py` — EventType literal union, _EVENT_CATEGORY_MAP, BaseEvent dataclass, specific event dataclasses with to_dict() methods, EventEmitter pub/sub class
- `src/policy_factory/server/broadcast.py` — BroadcastHandler that subscribes to EventEmitter, persists events, and broadcasts via WebSocket

Frontend event types:

- `ui/src/types/events.ts` — TypeScript EventType union, BaseEvent interface, specific event interfaces, PolicyEvent discriminated union, EVENT_CATEGORY_MAP

Example event patterns to follow:

- AgentTextChunk — streaming text event with chunk content
- LayerGenerationCompleted — completion event with result metadata
- CascadeQueued — notification event for queued work

## Reuse Opportunities

- Follow the exact dataclass pattern from existing events: inherit from BaseEvent (or use same field pattern), define event_type as class attribute, implement to_dict()
- Add to existing EventType literal union and _EVENT_CATEGORY_MAP
- Mirror the backend events 1:1 in TypeScript with matching interfaces
- Add to the PolicyEvent union for type-safe handling

## Deliverables

### Backend Event Definitions

Add a new "conversation" category and the following event types to events.py:

**conversation_started**
- Emitted when a conversation turn begins processing
- Fields: conversation_id

**conversation_text_chunk**
- Emitted during streaming response generation
- Fields: conversation_id, text (the chunk content)

**conversation_file_edit**
- Emitted when the agent writes or deletes a file
- Fields: conversation_id, layer_slug, filename, action ("write" or "delete")

**conversation_turn_complete**
- Emitted when the agent finishes its turn
- Fields: conversation_id, message_id, files_edited (list of layer_slug/filename paths)

**conversation_cascade_pending**
- Emitted when conversation edits to foundational layers queue a cascade
- Fields: conversation_id, starting_layer

**conversation_turn_error**
- Emitted when the agent turn fails
- Fields: conversation_id, error_message
