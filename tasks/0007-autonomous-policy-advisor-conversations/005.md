# 005: Conversation database schema

## Context

The conversation feature requires persistent storage for conversations and their messages. Each conversation belongs to either a specific item (layer_slug + filename) or to a layer as a whole (layer_slug only). Messages track the dialogue history including which files were edited during assistant turns.

This step creates the database tables and the ConversationStoreMixin with CRUD operations, following the existing mixin pattern used throughout the codebase.

## Deliverables

### Schema Additions

Add two new tables to the SCHEMA string in schema.py:

**conversations table**
- id: unique identifier (TEXT PRIMARY KEY)
- layer_slug: which layer this conversation is about (TEXT NOT NULL)
- filename: which item within the layer, or NULL for layer-level conversations (TEXT, nullable)
- created_at: when the conversation started (TEXT NOT NULL, ISO8601)
- last_active_at: when the last message was added (TEXT NOT NULL, ISO8601)
- Index on layer_slug and filename for efficient filtering

**messages table**
- id: unique identifier (TEXT PRIMARY KEY)
- conversation_id: foreign key to conversations (TEXT NOT NULL)
- role: either "user" or "assistant" (TEXT NOT NULL)
- content: the message text (TEXT NOT NULL)
- created_at: when the message was created (TEXT NOT NULL, ISO8601)
- files_edited: JSON array of edited file paths, or NULL for user messages (TEXT, nullable)
- Index on conversation_id for efficient message retrieval

### ConversationStoreMixin

Create a new store mixin file with:

**Dataclasses**
- Conversation: id, layer_slug, filename (optional), created_at, last_active_at
- Message: id, conversation_id, role, content, created_at, files_edited (optional list)

**CRUD Operations**
- create_conversation: creates a conversation for an item or layer, returns ID
- get_conversation: retrieves a conversation by ID with all fields
- list_conversations_for_item: returns conversations matching layer_slug and filename
- list_conversations_for_layer: returns conversations matching layer_slug where filename is NULL
- delete_conversation: removes a conversation and all its messages
- add_message: creates a message in a conversation, updates last_active_at
- get_messages: retrieves all messages for a conversation in chronological order

**Helper Methods**
- _row_to_conversation: converts sqlite3.Row to Conversation dataclass
- _row_to_message: converts sqlite3.Row to Message dataclass, parsing files_edited JSON

### Store Registration

Update PolicyStore in __init__.py to inherit from ConversationStoreMixin.
