# 016: Layer page sidebar integration

## Context

The LayerDetailPage needs the same conversation sidebar integration as ItemDetailPage. Layer-level conversations allow users to discuss the entire layer — the AI can add, edit, or remove items within it. The integration follows the same pattern as the item page: toggle button, layout adjustment, and content refresh on AI edits.

## Related Code

Existing page structure:

- `ui/src/pages/LayerDetailPage.tsx` — Current layer view with Page Header, Narrative Summary, Feedback Memos, Items List, Critic Assessment sections. Uses useLayerStore for data fetching and state management.
- `ui/src/pages/ItemDetailPage.tsx` — (from step 015) reference for sidebar integration pattern

Components to integrate:

- `ui/src/components/organisms/ConversationSidebar.tsx` — (from step 014) sidebar component

Store integration:

- `ui/src/stores/layerStore.ts` — fetchLayerDetail(), clearLayerDetail(), refresh() methods
- `ui/src/stores/conversationStore.ts` — (from step 012) for conversation state

## Reuse Opportunities

- Follow the exact same integration pattern from ItemDetailPage (step 015)
- Use the same toggle button placement and styling approach
- Use the same layout adjustment strategy
- Use the same content refresh mechanism via layerStore
- The ConversationSidebar component works for both items and layers — just pass filename as null for layer-level

## Deliverables

### Toggle Button

Add a conversation toggle button to the layer page header:
- Place alongside existing action buttons (Refresh, Version History)
- Use the same chat/message icon as ItemDetailPage
- Toggle opens/closes the sidebar
- Visual indicator when sidebar is open

### Sidebar State

Add local state for sidebar visibility:
- isConversationOpen boolean state
- Toggle function for the button
- Initialize conversationStore context with layerSlug and null filename when sidebar opens

### Layout Adjustment

Modify the page layout to accommodate the open sidebar:
- Follow the same approach used in ItemDetailPage
- Content shifts or narrows when sidebar is open
- Smooth transition on open/close
- All sections (narrative, memos, items list, critics) remain usable at narrower width

### Content Refresh

Wire up live content refresh when AI edits items within the layer:
- Subscribe to layerStore or listen for file edit events
- When a conversation_file_edit event affects the current layer, refetch layer data
- This includes changes to any item within the layer, the README.md narrative, or item additions/deletions
- Update the view with fresh content (items list, narrative summary)

### Sidebar Props

Pass correct props to ConversationSidebar:
- isOpen: from local sidebar state
- onClose: callback to close sidebar
- layerSlug: from URL params
- filename: null (indicating layer-level conversation)

### Layer-Level Conversation Behavior

The sidebar at layer level differs slightly from item level:
- Conversation discusses the layer as a whole
- AI can edit/add/remove multiple items within the layer
- AI can edit the README.md narrative
- Header shows layer name instead of item name
