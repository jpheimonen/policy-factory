# 004: Values prompt fix

## Context

The values generator prompt already contains extensive instructions for producing "controversial tension-pairs" with anti-consensus filters and requirements for genuine two-sidedness. However, the actual outputs don't follow these instructions — they produce single-topic consensus areas like "Social Welfare and Equality" instead of genuine tension-pairs like "Individual Liberty vs. Collective Welfare".

The problem is not that the instructions are missing, but that they're not being followed. The fix needs to be more aggressive: add concrete examples of good vs. bad outputs, enforce explicit formatting requirements, and strengthen the self-check mechanisms.

Additionally, the prompt needs to reference the new philosophy layer as grounding for what genuine tensions look like.

## Related Code

- `src/policy_factory/prompts/generators/values.md` — The main values generator prompt that needs strengthening

- `src/policy_factory/prompts/sections/anti-slop.md` — Writing standards that could inform the anti-consensus enforcement

## Deliverables

### Explicit Title Format Requirement

Add a mandatory title format section that requires:

- Titles must follow explicit "X vs. Y" format with the word "vs." separating the two priorities
- Single-topic titles are explicitly forbidden with examples of what NOT to do
- The two sides must represent genuinely opposing priorities, not two aspects of the same goal

### Good vs. Bad Examples Section

Add a calibration section showing concrete examples:

**Bad titles to reject:**
- "Social Welfare and Equality" — single topic, no tension
- "Environmental Sustainability and Climate Action" — two related goals, not opposing priorities
- "Democratic Institutions and Rule of Law" — consensus item, no controversy
- "Economic Prosperity and Competitiveness" — single-topic domain category

**Good titles to emulate:**
- "Individual Liberty vs. Collective Welfare" — genuine tradeoff
- "Ethnic & Cultural Cohesion vs. Open Immigration" — controversial, two-sided
- "Military Sovereignty vs. Alliance Dependence" — real dilemma with opposing pulls
- "Welfare Generosity vs. Work Incentives" — genuine policy tension

### Strengthened Anti-Consensus Filter

Enhance the existing anti-consensus filter with:

- A self-check step: before writing any item, the AI must explicitly verify the title follows "X vs. Y" format
- A controversy test that requires naming which political faction would be uncomfortable with each side
- Explicit instruction to delete and rewrite any item that fails the format check

### Philosophy Layer Reference

Add instruction to reference the philosophy layer:

- The philosophy layer's normative axioms define what genuine value tensions look like
- When in doubt about whether something is a real tension, check if it represents a conflict between foundational philosophical commitments identified in the philosophy layer

### Multi-Perspective Requirement

Strengthen the requirement for presenting opposing arguments:

- Each side of the tension must include arguments that a thoughtful advocate of that position would actually make
- Explicitly require arguments from different political traditions (conservative, progressive, libertarian, communitarian)
- Forbid framing one side as "the problem" and the other as "the solution"
