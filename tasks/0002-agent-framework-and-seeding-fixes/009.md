# Step 009: Update Data Initialization

## Objective

Modify `initialize_data_directory()` to no longer write pre-seeded values from `seed_values.py`. The values directory should start empty, with values created through explicit seeding.

## Changes Required

### src/policy_factory/data/init.py

Remove the values seeding from initialization:

```python
"""
Data directory initialization.
"""

from pathlib import Path

from policy_factory.data.git import init_data_repo, is_git_repo
from policy_factory.data.layers import LAYERS


def initialize_data_directory(data_dir: str) -> None:
    """
    Initialize the data directory structure.

    Creates:
    - Git repository (if not exists)
    - Layer directories
    - Placeholder README.md in each layer

    Does NOT seed any content — values and SA seeding are explicit operations.
    """
    data_path = Path(data_dir)

    # Create data directory if needed
    data_path.mkdir(parents=True, exist_ok=True)

    # Initialize git repo if not exists
    if not is_git_repo(data_dir):
        init_data_repo(data_dir)

    # Create layer directories and README placeholders
    for layer in LAYERS:
        layer_dir = data_path / layer.directory
        layer_dir.mkdir(exist_ok=True)

        readme_path = layer_dir / "README.md"
        if not readme_path.exists():
            readme_path.write_text(f"# {layer.name}\n\nThis layer is not yet populated.\n")

    # REMOVED: Writing pre-seeded values from seed_values.py
    # Values are now created through explicit POST /api/seed/values
```
