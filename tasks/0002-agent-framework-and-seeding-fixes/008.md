# Step 008: Fix SA Seed Endpoint

## Objective

Modify the existing SA seed endpoint to:
1. Remove the 409 "already seeded" guard — allow re-seeding at any time
2. Add an optional `context` field for human-provided situational context
3. Clear existing SA files before writing new ones

## Changes Required

### src/policy_factory/server/routers/seed.py

Update the existing SA seed endpoint:

```python
from pydantic import BaseModel
from typing import Optional


class SASeedRequest(BaseModel):
    """Request body for SA seeding."""
    context: Optional[str] = None


class SASeedResponse(BaseModel):
    success: bool
    items_created: int
    cascade_id: str | None
    message: str


@router.post("/", response_model=SASeedResponse)
async def seed_situational_awareness(
    request: SASeedRequest = SASeedRequest(),
    _user=Depends(require_auth),
    client=Depends(get_anthropic_client),
    data_dir=Depends(get_data_dir),
    event_emitter=Depends(get_event_emitter),
):
    """
    Seed the situational awareness layer.

    Clears existing SA items and creates new ones based on web research
    and any provided context.

    Can be called multiple times — each call replaces existing content.
    """
    # REMOVED: The 409 guard checking if SA layer already has items

    # Clear existing SA files
    existing = list_items("situational-awareness", data_dir)
    for item in existing:
        delete_item("situational-awareness", item["slug"], data_dir)

    # Load and customize the seed prompt
    base_prompt = load_prompt("seed/seed")

    if request.context:
        prompt = f"""Human-provided situational context:

{request.context}

---

{base_prompt}

When creating SA items, incorporate the human-provided context above alongside your web research. The human context represents the user's understanding of the current situation and should inform your analysis."""
    else:
        prompt = base_prompt

    # Create agent config with web search and file tools
    config = create_seed_config()

    # Run the agent
    session = AgentSession(
        config=config,
        event_emitter=event_emitter,
        context_id="sa-seed",
        agent_label="SA Seeder",
        client=client,
        data_dir=data_dir,
    )

    result = await session.run(prompt)

    if result.is_error:
        raise HTTPException(status_code=500, detail=result.error_message)

    # Commit the changes
    commit_changes(data_dir, "Seed situational awareness layer")

    # Count created items
    items = list_items("situational-awareness", data_dir)

    # Trigger cascade from SA layer
    cascade_id = await trigger_cascade_from_sa(event_emitter, data_dir, client)

    return SASeedResponse(
        success=True,
        items_created=len(items),
        cascade_id=cascade_id,
        message=f"Created {len(items)} SA items, cascade triggered",
    )
```

### Update seed prompt to handle context

The seed prompt should work with or without human context. The endpoint prepends context when provided, so the base prompt doesn't need changes.

## Key Changes from Current Implementation

1. **No 409 guard**: Previously returned 409 if `list_items("situational-awareness")` was non-empty. Now always proceeds.

2. **Clear before seed**: Delete all existing SA items before running the agent. This ensures a clean slate on re-seed.

3. **Optional context**: Accept a request body with optional `context` string. When provided, prepend to the prompt.

4. **Request model**: Change from no body to accepting `SASeedRequest` with optional context.
