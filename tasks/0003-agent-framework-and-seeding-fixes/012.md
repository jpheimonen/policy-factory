# 012: Rewrite agent session tests

## Context

The existing agent session tests mock the nonexistent `claude_agent_sdk` using elaborate helper classes (`MockTextBlock`, `MockAssistantMessage`, `MockResultMessage`) and `sys.modules` patching. With the switch to the Anthropic SDK, these tests need complete rewriting to mock the new SDK's streaming interface and event types.

The test scenarios themselves remain valid — we still need to verify retry logic, error handling, streaming behavior, and meditation filtering. Only the mocking infrastructure changes.

## Related Code

Existing files/patterns to reference or extend:

- `tests/test_agent_session.py` — The file to rewrite. Contains 465 lines of tests organized into test classes: `TestIsTransientError`, `TestAgentResult`, `TestAgentSessionInit`, `TestAgentSessionRun`, `TestAgentSessionRetry`, `TestAgentSessionConfig`. The test scenarios provide a good reference for what behaviors to verify.
- `tests/test_meditation_filter.py` — Tests for the meditation filter. Should remain unchanged since the filter itself isn't changing.
- `tests/conftest.py` — Contains shared fixtures. May need new fixtures for mocking Anthropic client.

## Reuse Opportunities

- Preserve the test class organization structure
- Preserve the test scenario coverage (retry, errors, streaming, config)
- The `_is_transient_error()` tests can largely stay the same since that function's behavior is similar
- `TestAgentResult` tests can stay the same since the dataclass isn't changing much
- Meditation filter integration tests need new mock infrastructure but same assertions

## Deliverables

- Rewrite `tests/test_agent_session.py` with new mocking infrastructure:
  - Create helpers to mock Anthropic SDK streaming events (text deltas, tool use blocks, message deltas)
  - Create mock async context manager for the streaming response
  - Remove old `MockTextBlock`, `MockAssistantMessage`, `MockResultMessage` classes
  - Remove `sys.modules` patching for `claude_agent_sdk`

- Preserve all test scenarios:
  - Single-turn conversation returns expected text
  - Multi-turn conversation with tool calls works correctly
  - Text chunks are emitted as AgentTextChunk events
  - Meditation content is filtered from emitted events but captured in full output
  - Rate limit errors trigger retry with backoff
  - Server errors trigger retry
  - Auth errors fail immediately
  - Context overflow raises ContextOverflowError
  - Max retries exceeded raises AgentError

- Add new test scenarios for Anthropic-specific behavior:
  - Tool results are correctly formatted and passed back
  - Unknown tool names return error to model
  - Conversation terminates on end_turn stop reason

- Add fixtures to conftest.py if shared mock infrastructure is useful across test files
