# 003: Rewrite AgentSession core

## Context

The `AgentSession` class currently wraps a nonexistent `claude_agent_sdk` package. The `_run_once()` method needs complete replacement with an implementation using the official Anthropic Python SDK. This is the core of the agent framework rewrite — it enables all agent-powered features (cascades, critics, heartbeat, seeding) to function.

The implementation must use a custom streaming agentic loop rather than the SDK's `tool_runner` beta, because:
- The application is async (FastAPI) and requires `AsyncAnthropic`
- Real-time token streaming to WebSocket must be preserved for the cascade viewer typewriter effect
- The `tool_runner` beta only yields completed messages, not streaming deltas

## Related Code

Existing files/patterns to reference or extend:

- `src/policy_factory/agent/session.py` — The file to modify. The `run()` method with retry logic and error classification should be preserved; only `_run_once()` needs rewriting. The `AgentResult` dataclass should be kept.
- `src/policy_factory/agent/meditation_filter.py` — `MeditationFilter` class that processes streamed text to suppress countdown content. Must continue to be used in the new implementation.
- `src/policy_factory/agent/errors.py` — `AgentError` and `ContextOverflowError` exceptions should continue to be raised in appropriate situations.
- `src/policy_factory/events.py` — `AgentTextChunk` event and `EventEmitter` for broadcasting streamed text. The new implementation must emit these events during streaming.
- `src/policy_factory/agent/tools.py` — The file tools module created in step 002; tool execution happens here.

## Reuse Opportunities

- Preserve the existing `run()` method structure with its retry loop and error classification logic
- Preserve `_is_transient_error()` function for classifying retryable errors
- Preserve `AgentResult` dataclass for returning results
- Reuse `MeditationFilter` unchanged — it operates on text chunks regardless of source
- Reuse existing event emission pattern via `EventEmitter.emit()`

## Deliverables

- Rewrite `_run_once()` method in `src/policy_factory/agent/session.py` to:
  - Use `AsyncAnthropic` client with `messages.stream()` for streaming responses
  - Implement an agentic loop that continues until the model signals end_turn without pending tool calls
  - Stream text deltas through the meditation filter and emit as `AgentTextChunk` events
  - Handle tool_use blocks by executing the appropriate tool and feeding results back
  - Capture full output including meditation content for the `AgentResult`

- Update `AgentSession.__init__()` to accept:
  - The shared `AsyncAnthropic` client instance (shared client setup happens in step 010; for this step, just define the interface)
  - The data directory path for file tool execution

- Update error detection to recognize Anthropic SDK error types:
  - Rate limit errors should trigger retry
  - Context length errors should raise `ContextOverflowError`
  - Authentication errors should fail immediately

## Acceptance Criteria

- [x] Single-turn conversations (no tool calls) return expected text in AgentResult
- [x] Multi-turn conversations with tool calls execute tools and continue until completion
- [x] Text deltas from model responses are emitted as AgentTextChunk events during streaming
- [x] Meditation filter suppresses countdown content from emitted events
- [x] Full output in AgentResult captures all text including meditation content
- [x] Rate limit errors (429) trigger retry with exponential backoff
- [x] Server errors (500, 502, 503, 529) trigger retry
- [x] Authentication errors fail immediately without retry
- [x] Context overflow errors raise ContextOverflowError without retry
- [x] Max retries exceeded raises AgentError with descriptive message
- [x] Conversation loop terminates when model signals end_turn without pending tool calls
- [x] Tool results are correctly formatted and passed back to the model
- [x] Unknown tool names return error result to model rather than crashing
- [x] AgentSession accepts client and data_dir in constructor
