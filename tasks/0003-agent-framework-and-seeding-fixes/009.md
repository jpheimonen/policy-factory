# 009: Update data initialization

## Context

Currently, `initialize_data_directory()` writes pre-seeded values from `seed_values.py` during first-run setup. These values are descriptive facts, not axiomatic values, and the content is wrong. With the new approach, values are populated via explicit seeding through the API endpoint, not at startup.

The initialization should create the directory structure and git repo but leave the values layer empty (aside from a README placeholder). Users explicitly trigger values seeding when ready.

## Related Code

Existing files/patterns to reference or extend:

- `src/policy_factory/data/init.py` — The file to modify. Contains `initialize_data_directory()` which calls `_write_seed_values()` to populate values at startup.
- `src/policy_factory/data/seed_values.py` — Contains `SEED_VALUES` list that is currently written at startup. This file can be deleted or deprecated after this change.

## Reuse Opportunities

- Keep all other initialization logic: git repo creation, layer directory creation, README placeholder writing
- The pattern of creating directories and placeholders remains the same, just without the values content

## Deliverables

- Modify `initialize_data_directory()` in `src/policy_factory/data/init.py`:
  - Remove the call to `_write_seed_values()`
  - Keep git repo initialization
  - Keep all five layer directory creation
  - Keep README.md placeholder creation in each layer
  - Keep the initial git commit (but with updated message reflecting no values)

- Either delete `src/policy_factory/data/seed_values.py` or add a deprecation notice:
  - Option A: Delete the file entirely since it's no longer used
  - Option B: Keep the file with a comment indicating it's deprecated and kept for reference only
  - Recommendation: Delete to avoid confusion

- Remove the `_write_seed_values()` helper function from init.py (no longer needed)

- Update any imports that reference `seed_values.py` if deleting the file

- [x] initialize_data_directory() creates all five layer directories
- [x] initialize_data_directory() creates README.md placeholders in each layer

- [x] Values directory contains only README.md after initialization
- [x] seed_values.py is deleted or clearly marked deprecated

- [x] Initialization remains idempotent (safe to call multiple times)
