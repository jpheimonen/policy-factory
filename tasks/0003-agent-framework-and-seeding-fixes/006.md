# 006: Create values seed prompt

## Context

The current values layer contains descriptive facts about Finland (e.g., "Finland joined NATO in 2023") rather than axiomatic values (e.g., "National survival takes precedence over ideological purity"). A new prompt is needed to instruct the agent to synthesize genuine axiomatic values from its knowledge of Finnish policy discourse.

Unlike the SA seed agent which uses web search, the values seed agent uses Claude's training knowledge — Finnish constitutional principles, cross-party consensus areas, and policy discourse are well within the training data. No tools are needed; this is a single-call analysis task.

## Related Code

Existing files/patterns to reference or extend:

- `src/policy_factory/prompts/seed/seed.md` — The SA seed prompt. Follow a similar structure but with different instructions focused on values rather than situational awareness.
- `src/policy_factory/prompts/generators/values.md` — The values layer generator prompt. Contains context about what values are and how they should be structured.
- `src/policy_factory/data/seed_values.py` — Contains the current (wrong) descriptive values. Useful to understand what NOT to produce and to see the expected output format (filename, title, body).

## Reuse Opportunities

- Follow the prompt file structure pattern from existing prompts in the seed/ and generators/ directories
- Reference the expected markdown output format with YAML frontmatter from existing prompts
- The prompt will be loaded via the existing `load_prompt()` function from the prompts module

## Deliverables

- New prompt file `src/policy_factory/prompts/seed/values.md` containing:
  - Clear instruction that values should be axiomatic/normative statements, not descriptive facts
  - Guidance on source material: Finnish constitutional principles, cross-party consensus, Nordic policy traditions, EU membership context, security policy evolution
  - Expected output format specification with explicit delimiter between values:
    - Each value is a complete markdown document with YAML frontmatter
    - Values are separated by the standard frontmatter delimiter `---` at the start of each new value
    - Frontmatter fields: `title` (string) and `tensions` (list of strings referencing other value titles)
    - Body follows the closing `---` of frontmatter
  - Request for tension identification: each value should list 2-3 other values it can conflict with
  - Target of 8-12 foundational values covering areas like sovereignty, security, social equality, rule of law, economic pragmatism, environmental responsibility, European integration, Nordic cooperation, education, technology
  - Explicit instruction that this uses Claude's knowledge — no web search or file tools
  - Instruction to produce all values in a single response with clear separation
