# 001: Add Anthropic SDK dependency

## Context

The agent framework currently references a nonexistent `claude_agent_sdk` package. Before any agent functionality can work, we need to add the official Anthropic Python SDK as a project dependency. This is a foundational step that unblocks all subsequent agent framework work.

## Related Code

Existing files/patterns to reference or extend:

- `pyproject.toml` — Project dependencies are listed in the `dependencies` array under `[project]`. The existing dependencies (fastapi, uvicorn, bcrypt, pyjwt, pyyaml, apscheduler) follow a pattern of specifying minimum versions with `>=`.

## Reuse Opportunities

- Follow the existing dependency specification pattern in pyproject.toml
- No code reuse needed for this step — it's purely a dependency addition

## Deliverables

- Add the `anthropic` package to project dependencies in pyproject.toml
- Verify the package installs correctly and can be imported
- Ensure compatibility with the existing Python version constraint (>=3.10)

## Acceptance Criteria

- [ ] The `anthropic` package is listed in pyproject.toml dependencies
- [ ] Running `uv sync` completes without errors
- [ ] The anthropic module can be imported successfully in Python
- [ ] The AsyncAnthropic client class can be imported from the anthropic package
- [ ] No existing tests are broken by adding this dependency
