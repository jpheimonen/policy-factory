# 013: Add file tools tests

## Context

The file tools module created in step 002 provides critical security functionality — sandbox validation that prevents agents from accessing files outside the data directory. This needs thorough testing to ensure path traversal attacks are blocked and all tool behaviors work correctly.

## Related Code

Existing files/patterns to reference or extend:

- `tests/test_data_layers.py` — Tests for the layers.py module. Similar pattern of testing file operations with temporary directories.
- `tests/test_data_markdown.py` — Tests for markdown reading/writing. Shows pattern for testing file I/O.
- `tests/conftest.py` — Contains fixtures including `tmp_path` usage patterns for temporary test directories.
- `src/policy_factory/agent/tools.py` — The module being tested (created in step 002).

## Reuse Opportunities

- Use pytest's built-in `tmp_path` fixture for creating isolated test directories
- Follow the test organization pattern from other data layer tests
- Similar file operation assertions can be adapted from test_data_layers.py

## Deliverables

- New test file `tests/test_agent_tools.py` containing:

  - Path validation tests:
    - Valid paths within data directory are accepted
    - Paths with `../` that escape sandbox are rejected
    - Double traversal paths are rejected
    - Absolute paths outside sandbox are rejected
    - Symlinks that resolve outside sandbox are rejected

  - list_files tests:
    - Returns markdown filenames in directory
    - Excludes README.md from results
    - Returns empty list for empty directory
    - Returns error for paths outside sandbox
    - Returns error when path is not a directory

  - read_file tests:
    - Returns content for existing file
    - Returns error for nonexistent file
    - Returns error for paths outside sandbox

  - write_file tests:
    - Creates new file with content
    - Overwrites existing file
    - Creates parent directories if needed
    - Returns error for paths outside sandbox

  - delete_file tests:
    - Removes existing file
    - Succeeds silently for nonexistent file (idempotent)
    - Returns error for paths outside sandbox

  - Tool definition tests:
    - FILE_TOOLS contains all four tools
    - READ_ONLY_TOOLS contains only list_files and read_file
    - All tools have required schema fields

## Acceptance Criteria

- [ ] Test file exists at tests/test_agent_tools.py
- [ ] Path validation tests cover all escape vectors (../, absolute paths, symlinks)
- [ ] All list_files behaviors are tested
- [ ] All read_file behaviors are tested
- [ ] All write_file behaviors are tested
- [ ] All delete_file behaviors are tested
- [ ] Tool definition structure tests verify required fields
- [ ] Tests use tmp_path fixture for isolation
- [ ] All tests pass
- [ ] No security-relevant edge cases are missed
