# 007: Implement values seed endpoint

## Context

Currently there is no endpoint to seed the values layer — only SA seeding exists. The values layer was previously populated at startup with hardcoded descriptive content. With the new approach, values seeding is an explicit operation triggered via API, using an agent to synthesize axiomatic values from Claude's knowledge.

The values seed agent is unique in that it needs no tools — it produces values based on training data, not web search or file access. The endpoint receives the agent's output, parses it into individual value documents, and writes them to the data/values/ directory.

## Related Code

Existing files/patterns to reference or extend:

- `src/policy_factory/server/routers/seed.py` — Contains the existing SA seed endpoint. The new values endpoint should be added to this same router, following the same patterns for authentication, agent execution, and response structure.
- `src/policy_factory/data/layers.py` — Contains `list_items()`, `write_item()`, `delete_item()` for file operations. Use these to manage values files.
- `src/policy_factory/data/git.py` — Contains `commit_changes()` for committing after writes.
- `src/policy_factory/agent/session.py` — `AgentSession` for running the agent.
- `src/policy_factory/agent/config.py` — Need to create config with values-seed role.

## Reuse Opportunities

- Follow the existing SA seed endpoint pattern for structure, authentication, and error handling
- Use existing `list_items()` and `delete_item()` to clear existing values before writing
- Use existing `write_item()` or `write_markdown()` to create value files
- Use existing `commit_changes()` for git commit
- Use existing Pydantic response model pattern from the seed router

## Deliverables

- New endpoint `POST /api/seed/values` in the seed router:
  - Requires authentication (same as SA seed)
  - Loads the values seed prompt created in step 006
  - Creates agent config with values-seed role (no tools)
  - Runs the agent and captures output
  - Parses the output to extract individual value documents (frontmatter + body)
  - Clears existing values files before writing new ones
  - Writes each value as a markdown file with proper frontmatter
  - Commits changes to git
  - Returns success status and count of values created

- Response model for the endpoint including success boolean and values_created count

- Output parsing logic to extract individual values from the agent's response:
  - Split on frontmatter delimiters: each value starts with `---` followed by YAML, then `---`, then body
  - For each block, parse YAML frontmatter to extract `title` and `tensions` fields
  - Slugify the title to generate filename (lowercase, hyphens for spaces, strip special chars)
  - Extract body text following the closing `---`
  - Skip malformed blocks that lack required fields rather than failing entire operation
  - Return error only if zero values could be parsed (partial success is acceptable)
