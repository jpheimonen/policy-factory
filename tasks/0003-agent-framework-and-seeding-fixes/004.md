# 004: Update AgentConfig for tool configuration

## Context

The `AgentConfig` dataclass currently contains fields specific to the nonexistent Claude Code SDK (`cwd`, `max_turns`, `permission_mode`). These need to be removed. Additionally, different agent types need different tools — generators need file write tools, critics need read-only tools, heartbeat needs web search, and the values seeder needs no tools at all. This step updates the configuration to support per-agent tool sets.

## Related Code

Existing files/patterns to reference or extend:

- `src/policy_factory/agent/config.py` — The file to modify. Contains `AgentConfig` dataclass, `resolve_model()` function, `_DEFAULT_MODELS` and `_ENV_VAR_MAP` dicts, and `AgentRole` type literal. The model resolution pattern should be preserved and extended.
- `src/policy_factory/agent/tools.py` — The file tools module created in step 002. Contains tool definitions that need to be referenced here.

## Reuse Opportunities

- Preserve the existing `resolve_model()` function and its environment variable override pattern
- Extend `_DEFAULT_MODELS` and `_ENV_VAR_MAP` with the new values-seed role
- Follow the same pattern of role-based configuration for tool assignment
- Keep the dataclass pattern for `AgentConfig`

## Deliverables

- Update `AgentConfig` dataclass in `src/policy_factory/agent/config.py`:
  - Remove `cwd` field (no longer applicable with Anthropic SDK)
  - Remove `max_turns` field (no longer applicable)
  - Remove `permission_mode` field (no longer applicable)
  - Keep `model` and `system_prompt` fields
  - Add a `tools` field that holds the list of tool definitions for this agent

- Add `values-seed` to the supported agent roles:
  - Add to `AgentRole` type literal
  - Add default model mapping (use same tier as other seed operations)
  - Add environment variable mapping for model override

- Create a tool configuration mapping that returns appropriate tools per role:
  - `generator` — full file tools (list, read, write, delete)
  - `critic` — read-only tools (list, read)
  - `synthesis` — no tools
  - `classifier` — no tools
  - `heartbeat-skim` — web search only (Anthropic's server-side tool)
  - `heartbeat-triage` — web search only (Anthropic's server-side tool)
  - `heartbeat-sa-update` — file tools plus web search
  - `seed` — file tools plus web search
  - `values-seed` — no tools (uses Claude's knowledge)
  - `idea-evaluator` — no tools
  - `idea-generator` — no tools

- Web search tool configuration:
  - Web search uses Anthropic's built-in server-side tool `web_search_20250305`
  - Server-side tools are specified differently than custom tools — they use a `type` field rather than a full tool definition
  - The tool is included in the tools array as a type reference, and Anthropic's API handles execution automatically (no client-side tool execution needed)
  - The agentic loop in AgentSession does not need to handle web_search tool_use blocks — results come back automatically from the API

- Add a helper function to resolve tools for a given role, similar to `resolve_model()`

## Acceptance Criteria

- [ ] AgentConfig no longer has cwd, max_turns, or permission_mode fields
- [ ] AgentConfig has a tools field for tool definitions
- [ ] resolve_model returns appropriate default for values-seed role
- [ ] resolve_model respects environment variable override for values-seed role
- [ ] Tool configuration returns file tools for generator role
- [ ] Tool configuration returns read-only tools for critic role
- [ ] Tool configuration returns empty list for synthesis and classifier roles
- [ ] Tool configuration returns Anthropic server-side web_search_20250305 tool for heartbeat-skim and heartbeat-triage roles
- [ ] Tool configuration returns file tools plus web search for heartbeat-sa-update and seed roles
- [ ] Tool configuration returns empty list for values-seed role
- [ ] Tool configuration returns empty list for idea-evaluator and idea-generator roles
- [ ] Existing callers of AgentConfig continue to work (may need updates in later steps)
