# 002: Create file tools module

## Context

With the Anthropic SDK, agents no longer have implicit filesystem access like Claude Code sessions did. Instead, agents interact with files through explicit tool calls. This step creates a new module providing file operation tools that agents can invoke, with strict sandbox validation to ensure agents can only access files within the data directory.

## Related Code

Existing files/patterns to reference or extend:

- `src/policy_factory/data/layers.py` — Contains `list_items()`, `read_item()`, `write_item()`, `delete_item()` functions that perform similar file operations on layer directories. These can be leveraged or wrapped by the new tools.
- `src/policy_factory/data/markdown.py` — Contains `read_markdown()` and `write_markdown()` for frontmatter handling. The file tools may use these internally.
- `src/policy_factory/agent/` — The new tools module should live here alongside session.py and config.py, following the existing module organization pattern.

## Reuse Opportunities

- The existing `layers.py` utilities handle the actual file I/O with proper frontmatter parsing; the new tools can delegate to these where appropriate
- Path validation logic should be centralized in a single helper function that all tools use
- Tool definition structures should follow the Anthropic API's expected format for tool schemas

## Deliverables

- New file `src/policy_factory/agent/tools.py` containing:
  - Path validation helper that ensures paths resolve within the data directory
  - `list_files` tool function that returns markdown filenames in a directory (excluding README.md)
  - `read_file` tool function that returns file content
  - `write_file` tool function that creates or overwrites a file
  - `delete_file` tool function that removes a file (idempotent)
  - Tool definition structures in Anthropic API format
  - A read-only subset of tools for critic agents
- Update `src/policy_factory/agent/__init__.py` to export the new tools

## Acceptance Criteria

- [x] Path validation accepts paths that resolve within the data directory
- [x] Path validation rejects paths containing `../` that would escape the sandbox
- [x] Path validation rejects absolute paths outside the data directory
- [x] Path validation rejects symlinks that resolve outside the sandbox
- [x] list_files returns markdown filenames excluding README.md
- [x] list_files returns empty list for empty directory
- [x] list_files returns error dict for paths outside sandbox
- [x] list_files returns error dict when path is not a directory
- [x] read_file returns file content for existing files
- [x] read_file returns error dict for nonexistent files
- [x] read_file returns error dict for paths outside sandbox
- [x] write_file creates new files with provided content
- [x] write_file overwrites existing files
- [x] write_file creates parent directories if needed
- [x] write_file returns error dict for paths outside sandbox
- [x] delete_file removes existing files
- [x] delete_file succeeds silently for nonexistent files
- [x] delete_file returns error dict for paths outside sandbox
- [x] Tool definitions include name, description, and input_schema fields
- [x] Read-only tool set contains only list_files and read_file
