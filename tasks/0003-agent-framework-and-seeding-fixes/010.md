# 010: Create shared Anthropic client

## Context

The Anthropic SDK client should be shared across all agent sessions rather than created fresh for each request. This provides connection pooling for concurrent agent runs (especially important during cascade when 6 critics run in parallel) and allows validation of the API key at startup.

The client needs to be initialized during FastAPI lifespan startup and made available to request handlers via dependency injection.

## Related Code

Existing files/patterns to reference or extend:

- `src/policy_factory/server/app.py` or `src/policy_factory/main.py` — Where FastAPI app is created and lifespan handlers are defined. The client should be initialized here.
- `src/policy_factory/server/deps.py` — Contains existing dependency injection functions like `get_current_user()`, `get_data_dir()`, `get_event_emitter()`, `get_store()`. The new client dependency should follow this pattern.
- `src/policy_factory/data/init.py` — Contains `get_data_dir()` which is called during startup. Similar pattern for environment-based configuration.

## Reuse Opportunities

- Follow the existing dependency injection pattern in deps.py
- Follow the existing lifespan/startup pattern for initializing shared resources
- The `get_data_dir()` pattern of checking environment variables with fallback is similar to how API key handling should work

## Deliverables

- Initialize AsyncAnthropic client during FastAPI lifespan startup:
  - Read ANTHROPIC_API_KEY from environment
  - If key is missing, log a warning but don't crash (app should start, agent operations will fail with clear error)
  - Create the client instance and store it for later access
  - Close the client properly during shutdown

- Create dependency function for accessing the client:
  - Add to deps.py following existing patterns
  - Returns the shared client instance
  - Raises clear error if client was not initialized (missing API key)

- Update all places that create AgentSession to use the shared client:
  - Seed router endpoints
  - Cascade orchestrator
  - Critic runner
  - Heartbeat orchestrator
  - Ideas evaluator/generator
  - Any other agent callers

## Acceptance Criteria

- [ ] AsyncAnthropic client is created during FastAPI startup
- [ ] Missing ANTHROPIC_API_KEY logs warning but does not crash startup
- [ ] Client is properly closed during FastAPI shutdown
- [ ] Dependency function returns the shared client instance
- [ ] Dependency function raises clear error if API key was missing
- [ ] All AgentSession instantiations use the shared client via dependency injection
- [ ] Concurrent agent sessions share the same client (connection pooling)
- [ ] App starts successfully without API key configured (for development/testing)
- [ ] Agent operations with missing API key return clear error message
